## 1. 大原則・経緯・現況の確認

### 1.1 開発・実装の大原則（最重要）

**要約定義書の大原則を遵守する：**

1. **根本解決 > 暫定解決**
   - 管理画面・ゲスト画面の確認で問題が発見された場合、根本原因を解決する
   - 一時的な回避策ではなく、設計レベルでの解決を優先

2. **シンプル構造 > 複雑構造**
   - Phase 2のステップ計画はシンプルで明確に
   - 過度に複雑な手順を避け、理解しやすく実行しやすい構造

3. **統一・同一化 > 特殊独自**
   - 既存のドキュメント（要約定義書、アーキテクチャ設計書）に準拠
   - 特殊な手順を避け、標準的なアプローチを採用

4. **具体的 > 一般**
   - 抽象的な説明ではなく、具体的な手順や確認項目を明確に
   - 実行可能な具体的な内容を記載

5. **拙速 < 安全確実**
   - 動作確認をしっかり行い、安全に進める
   - テストを省略せず、十分な検証を行ってから次のステップに進む

**この大原則は、Phase 2のすべてのステップにおいて最優先で遵守する。**
