## 残存課題（実行中に検知・2026-03-29）

### 管理画面「請求履歴・領収書」の金額表示（JPY）

- **現象（修正前）**: ステージング管理画面 **`/admin/billing`** の請求履歴テーブルで、Stripe Invoice の金額が **「¥39.8」** のように **100 分の 1** で表示されていた。
- **正**: 同一請求の Stripe ホスト済み請求ページでは **¥3,980** と一致させる。
- **原因**: JPY は Stripe 上 **ゼロデシマル通貨**のため `amount_due` は **円単位**。`PlanBilling.vue` で一律 `/ 100` していた。
- **対応（2026-03-30・実装済）**: **`GET /admin/invoices`** 応答に **`currency`**（Stripe Invoice と同じ）を追加し、フロントで [Stripe ゼロデシマル一覧](https://docs.stripe.com/currencies#zero-decimal) に基づき **最小単位→表示額** を変換。**変更前バックアップ**: `backups/20260330_planbilling_jpy_fix_pre/`。**検証**: `frontend` で `npm run lint` **exit 0**、`vue-tsc` 通過。
- **未了（要 Owner）**: **`develop` デプロイ後**のステージング **`/admin/billing`** で金額が **¥3,980** 等となっていることの目視確認（本番反映は別判断）。

### 請求書・領収書への消費税（内税 10%）表示（2026-03-30 検知）

- **指摘**: インボイス制度下で請求書・領収書に求められる **消費税・税率（内税 10% 等）の明示**が、現状の **Stripe ホスト済み請求ページ・ダウンロード領収書 PDF** では、**税額・税率が利用者期待どおりに見えない／不足している**（例: 領収書 PDF `Receipt-2495-9095.pdf`、請求 **FCTZEF69-0001**。金額は **￥3,980** と整合するが、**消費税の内訳表示は別課題**）。
- **リスク**: 適格請求書・適格簡易請求書としての **記載要件**に関する **コンプライアンス**。最終判断・設定方針は **税理士等の専門家**と **`docs/領収証_インボイス対応化_設定計画.md`** に基づき実施する。
- **対応**: Stripe 側の **税計算・請求書テンプレート・インボイス設定**、および本計画書の手順見直し。※本項は **法解釈の確定を記録するものではない**（運用上の未対応・課題のファクト記録）。
