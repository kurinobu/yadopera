### 3.3 CSV一括登録 有料オプション（開発者による代行サービス）

**概要**: 宿泊施設管理者が**自身で**CSVをアップロードする場合は無料。**開発者（運営）へFAQ登録を依頼する場合は有料オプション**。料金体系・対象プラン・料金に含まれる／含まれない内容は plan2.txt（CSV一括登録 実装計画書 v2.0）を正とする。代行契約時は運営が「CSV作成」**と**「管理画面への登録作業」の両方を履行する（CSVだけ渡してユーザーにアップロードしてもらうのは代行契約の履行ではない）。

**計画に含める内容**
- **オプション提供の運用開始**: 料金表の確定（plan2.txt 【料金体系】）、代行履行手順の整備・周知、契約フロー（見積・受注・履行・請求）。  
- **履行手段**: 方法C（バックエンドスクリプト `bulk_upload_faq_csv_for_facility.py`）を主推奨、方法B（依頼元ユーザーのJWTでAPI呼び出し）を補助。方法A（ユーザーにCSVを渡して自分でアップロード）は代行契約時には使わない。  
- **請求**: プラン料金とは別料金。Stripe実装後はオプション料金の請求連携を継承（plan2.txt）。Stripe未実装時は見積・請求は手動等で対応。

**参照**
- [開発者によるCSV一括登録代行_手順と方法_20260212.md](../CSV_bulk_registration_function_implementation_plan/開発者によるCSV一括登録代行_手順と方法_20260212.md) — 代行履行の手順・方法C/B/A・料金体系の引き継ぎ
- [CSV_bulk_registration_function_implementation_plan2.txt](../CSV_bulk_registration_function_implementation_plan/CSV_bulk_registration_function_implementation_plan2.txt) — ビジネス要件・料金体系・料金に含まれる／含まれない内容の正本

### 3.6 料金プランのアップグレード・ダウングレード機能

**概要**: 宿泊施設管理者が自分で料金プラン（Free/Mini/Small/Standard/Premium）をアップグレードまたはダウングレードする機能。現状は管理画面・APIにプラン変更の項目がなく、変更するにはDBの直接修正や運用スクリプト（例: `fix_existing_facilities_plan_settings.py`）による更新が必要。

**推奨**: **Stripe実装と同時に実装するのがよい**。サブスクリプション変更・請求サイクル・従量課金との整合を一括で設計・実装できるため。

**主な検討事項**  
- 管理画面に「プラン変更」画面または設定項目を追加（対象プラン一覧・料金・変更ボタン）。  
- API: 施設の `plan_type` / `subscription_plan` および `monthly_question_limit` / `faq_limit` / `language_limit` を更新するエンドポイント（Stripeサブスク変更と連携）。  
- プラン変更時の既存FAQの扱い（要約定義書・Phase2計画で「検討すること」と記載）。既存QRコード・既存FAQはプラン変更後も継続利用可能（同一施設のため）。  
- `plan_updated_at` の更新、請求期間（`plan_started_at`）との関係の整理。

**依存**: Stripe実装（サブスクリプション・プラン切り替え）と同時実施が望ましい。  
**参照**: [料金プラン変更とQR・FAQ継続利用_調査報告_20260212.md](../料金プラン変更とQR・FAQ継続利用_調査報告_20260212.md)、要約定義書 9章 収益モデル、Phase2_実装ステップ計画「プラン変更時の既存FAQの扱いを検討すること」。本ドキュメント 3.6 節。
