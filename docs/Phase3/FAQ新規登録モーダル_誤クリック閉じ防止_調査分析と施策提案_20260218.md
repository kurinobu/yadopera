# FAQ新規登録モーダル 誤クリック閉じ防止 — 調査分析と施策提案

**作成日**: 2026年2月18日  
**目的**: PoC運用中の宿泊施設管理者からの声「FAQ新規登録モーダル入力中に外を誤クリックするとモーダルが閉じ、入力途中の文面が消える」を予防する施策を、要約定義書・Phase3計画・大原則に準拠して提案する。**指示があるまでコード修正は行わない。**

**参照**: 要約定義書 v0.3、アーキテクチャ設計書 v0.3、Phase3 現況と残存課題（`Phase3_現況と残存課題_実装計画_20260209.md`）

## 0. 前提：目的・全体像・経緯・現況・大原則

### 0.1 目的・全体像（要約定義書・Phase3 より）

- **やどぺら**: 小規模宿泊施設向け外国人ゲスト対応自動化SaaS。QRコードでAIが24時間多言語自動応答。
- **Phase 3**: PoC実施（2ヶ月）**実行中**。3施設での運用・検証、バグ修正・機能改善、成功事例の記録が中心。
- **宿泊施設管理者向けFAQ**: Phase 2 で実装済み。FAQ管理画面で新規追加・編集・削除・CSV一括登録・改善提案などが利用可能。

### 0.2 経緯（ユーザー声）

- **事象**: PCでFAQ管理画面において、FAQ新規登録モーダルで入力している最中、モーダル以外の場所（オーバーレイ・背景）を誤ってクリックすると、モーダルが閉じる。その結果、**書き込み途中の質問文・回答文がすべて失われる**。
- **影響**: 単純な操作ミスだが、ユーザーにとって精神的ダメージが大きい。再入力の手間とストレスを防ぎたい。

### 0.3 現況（コードベース調査結果）

| 項目 | 現状 |
|------|------|
| **FAQ追加・編集モーダル** | `frontend/src/views/admin/FaqManagement.vue` で `<Modal v-model="showAddForm" ... @close="handleCloseForm">` を使用。タイトルは「FAQ編集」／「FAQ追加」。 |
| **Modal コンポーネント** | `frontend/src/components/common/Modal.vue`。**`closeOnBackdrop`**（デフォルト `true`）により、**オーバーレイ（背景）クリックで即座に `handleClose()` が呼ばれ、モーダルが閉じる**。 |
| **閉じる経路** | (1) オーバーレイ・外側クリック → `handleBackdropClick` → `closeOnBackdrop` が true のため `handleClose()`。(2) ヘッダーの閉じるボタン（×）→ `handleClose()`。(3) ESCキー → `handleEscape` → `handleClose()`。(4) FaqForm 内の「キャンセル」→ `emit('cancel')` → 親の `handleCloseForm()`。 |
| **FaqForm** | `frontend/src/components/admin/FaqForm.vue`。`formData`（翻訳リスト：質問・回答・言語・カテゴリ・優先度・intent_key）を保持。**「下書きありか」を表す isDirty / hasDraft は現状未公開**。 |
| **データ損失** | モーダルが閉じると `handleCloseForm()` で `showAddForm = false`, `editingFaq = null` となり、FaqForm はアンマウントされる。**入力内容は保持されない。** |

**結論**: 外側クリック・ESC・×ボタンのいずれも、**確認なしでモーダルが閉じる**設計のため、入力途中の誤操作で内容が失われている。

### 0.4 大原則（要約定義書 0. 節より・遵守する）

1. **根本解決 > 暫定解決** … 誤クリックで閉じる仕様を変え、下書きがあるときは「保存かキャンセルを選ばせる」ようにする。
2. **シンプル構造 > 複雑構造** … 既存の `Modal` の `closeOnBackdrop` や新規の「閉じる前コールバック」で完結させ、余計なレイヤーを増やさない。
3. **統一・同一化 > 特殊独自** … 共通 `Modal` の拡張（オプション）で対応し、FAQ専用の特殊コンポーネントを乱立させない。
4. **具体的 > 一般** … どの画面で・どの条件で警告を出すか、文言をどうするかを具体的に決める。
5. **拙速 < 安全確実** … 変更後は Docker 環境での動作確認・ステージングでのブラウザ確認を省略しない。
6. **Docker環境必須** … 修正・テストはすべて Docker 環境で実施する。

## 1. コードベース詳細（該当箇所）

### 1.1 Modal.vue（抜粋）

- **Props**: `modelValue`, `title`, `size`, `closable`（デフォルト true）, **`closeOnBackdrop`**（デフォルト **true**）。
- **Backdrop クリック**: ルートの `@click.self="handleBackdropClick"` とオーバーレイの `@click="handleBackdropClick"`。`handleBackdropClick` 内で `if (props.closeOnBackdrop) handleClose()`。
- **ESC**: `handleEscape` で `event.key === 'Escape' && props.modelValue && props.closable` のとき `handleClose()`。
- **現状**: 「閉じる前の確認」や「下書きあり時は閉じない」のフックは**存在しない**。

### 1.2 FaqManagement.vue（抜粋）

- **FAQ追加・編集モーダル**（86–98行）:
  - `<Modal v-model="showAddForm" :title="..." size="lg" @close="handleCloseForm">`。**`closeOnBackdrop` は未指定のため true**。
  - スロット内に `<FaqForm :faq="editingFaq" @submit="handleSubmitFaq" @cancel="handleCloseForm" />`。
- **handleCloseForm**: `showAddForm = false`, `editingFaq = null` のみ。確認ダイアログなし。

### 1.3 FaqForm.vue（抜粋）

- **formData**: `category`, `intent_key`, `translations[]`（各 `language`, `question`, `answer`）, `priority`, `is_active`。
- **編集時**: `watch(() => props.faq, ...)` で `formData` を初期化。
- **新規時**: `translations: [{ language: 'en', question: '', answer: '' }]`, `category: ''`, `priority: 3` 等。
- **公開API**: `defineEmits` は `submit`, `cancel` のみ。**「下書きあり」に相当する isDirty / hasDraft は未定義・未公開。**

## 2. 施策の方向性（大原則に沿った方針）

- **望ましい挙動**  
  - モーダル**外側クリック**または**ESC**で「閉じようとした」とき、**入力途中（下書きあり）の場合はモーダルを閉じず**、「保存するか、キャンセルをクリックしてください」の旨の**警告を表示**する。  
  - **保存**または**キャンセル**を明示的に押したときは、現行どおり閉じてよい（キャンセル時は内容破棄でよい）。
- **×ボタン**  
  - 外側クリック・ESCと同様に扱い、**下書きありのときは閉じず警告**とするのが一貫している（「閉じる」＝キャンセルに近いため、同じガードをかける）。

これにより、

- **根本解決**: 誤クリックで閉じる仕様をやめ、下書きがあるときは必ず「保存 or キャンセル」で閉じる。
- **シンプル**: 既存 Modal に「閉じる前コールバック」を追加し、FAQ側では「下書きありか」と「警告表示」だけを担当。
- **統一**: 他モーダル（CSV一括・無視確認など）は従来どおり `closeOnBackdrop` のまま利用可能。FAQ追加・編集モーダルのみオプションでガードをかける。

## 3. 施策提案（実装レベル）

### 3.1 Modal の拡張（統一・同一化）

- **新規オプション**: `beforeClose?: (source: 'backdrop' | 'escape' | 'button') => boolean | Promise<boolean>`
  - **意味**: 閉じる直前に呼ぶ。`false`（または Promise で resolve(false)）を返したら**閉じない**。`true` または未指定なら従来どおり閉じる。
  - **呼び timing**:  
    - オーバーレイクリック時: `handleBackdropClick` 内で、`closeOnBackdrop` の分岐の**前**に `beforeClose('backdrop')` を実行。  
    - ESC 時: `handleEscape` 内で `handleClose()` の**前**に `beforeClose('escape')` を実行。  
    - ×ボタン: 同様に `handleClose()` の前に `beforeClose('button')` を実行（オプションで。未指定ならそのまま閉じるでも可）。
  - **既存利用箇所**: `beforeClose` を渡していないモーダルは従来どおり。**後方互換**を保つ。

これで「閉じる前に親が介入できる」ようにし、FAQ側で「下書きありなら false を返し、警告表示」を実装する。

### 3.2 FaqForm の拡張（下書き検知）

- **下書きの定義（案）**:  
  - いずれかが真なら「下書きあり」:  
    - `translations` のいずれかで `question.trim()` または `answer.trim()` が空でない  
    - または `category !== ''`  
    - または `intent_key` が空でない  
  - 新規で何も触っていない初期状態（1言語・空・category 未選択）は「下書きなし」とする。
- **公開方法**:  
  - **案A**: `isDirty` を computed で定義し、`defineExpose({ isDirty })` で公開。親は `ref` で FaqForm を参照し、`faqFormRef.value?.isDirty` で判定。  
  - **案B**: `v-model:dirty="formDirty"` のように双方向で親に伝える。  
  - **推奨**: **案A**。既存の `@submit` / `@cancel` のパターンと相性が良く、Modal の `beforeClose` で参照するだけなのでシンプル。

### 3.3 FaqManagement の変更

- **FaqForm への ref**: `<FaqForm ref="faqFormRef" ... />` を追加。
- **Modal の利用**:
  - `beforeClose` を渡す:  
    `beforeClose(source) { if (faqFormRef.value?.isDirty) { showBackdropWarning.value = true; return false; } return true; }`
  - 下書きありで外側クリック／ESC／×を押したときは **閉じない** かつ **`showBackdropWarning = true`** とする。
- **警告表示**:
  - モーダル内（FaqForm の上またはヘッダー直下）に、`showBackdropWarning` が true のときだけ表示するバナーを追加。  
  - **文言（案）**: 「入力内容を残すには「作成」（または「更新」）を、破棄するには「キャンセル」をクリックしてください。」  
  - ユーザーが「作成」または「キャンセル」をクリックしたら `showBackdropWarning = false` にし、通常の閉じ処理に任せる。
- **closeOnBackdrop**:  
  - 常に `true` のままでよい。`beforeClose` が false を返す間は実際には閉じない。  
  - あるいは「下書きありのときは閉じない」を明示的にするため、`closeOnBackdrop` は true のままにし、閉じる可否の判断は `beforeClose` に一本化する（推奨）。

### 3.4 挙動まとめ

| 操作 | 下書きなし | 下書きあり |
|------|------------|------------|
| 外側クリック | モーダル閉じる | 閉じない ＋ 警告バナー表示 |
| ESC | モーダル閉じる | 閉じない ＋ 警告バナー表示 |
| ×ボタン | モーダル閉じる | 閉じない ＋ 警告バナー表示 |
| キャンセル | モーダル閉じる（破棄） | モーダル閉じる（破棄） |
| 保存/作成/更新 | モーダル閉じる（保存） | モーダル閉じる（保存） |

「保存かキャンセルをクリックしてください」は、上記バナー文言で実現する。

### 3.5 他モーダルへの影響

- **CSV一括登録モーダル**・**無視確認モーダル**: `beforeClose` を渡していないため、**従来どおり**オーバーレイ・ESCで閉じる。変更不要。
- **統一・同一化**: 共通 `Modal` のオプション追加のみで、既存利用箇所はそのまま。

## 4. 実装ステップ（案・修正は行わず計画のみ）

1. **Modal.vue**  
   - `beforeClose` プロパティを追加（オプション）。  
   - `handleBackdropClick` / `handleEscape` / ヘッダー閉じるボタンで、閉じる前に `beforeClose(source)` を実行。  
   - 戻りが `false` のときは `handleClose()` を呼ばない。Promise 対応する場合は `await` で扱う。

2. **FaqForm.vue**  
   - 上記条件で `isDirty` を computed で定義。  
   - `defineExpose({ isDirty })` で公開。

3. **FaqManagement.vue**  
   - FaqForm に `ref="faqFormRef"` を付与。  
   - `showBackdropWarning` を ref で保持。  
   - Modal に `:before-close="handleBeforeClose"` を渡し、`faqFormRef.value?.isDirty` が true のときは `showBackdropWarning = true` として `return false`。  
   - モーダル内に警告バナーを追加（`showBackdropWarning` が true のとき表示）。  
   - 保存成功時・キャンセルクリック時に `showBackdropWarning = false` を設定。

4. **確認**  
   - Docker 環境でフロント起動し、FAQ管理画面で「新規FAQ追加」→ 入力途中で外側クリック・ESC・×を押すとモーダルが閉じず警告が出ること、保存・キャンセルで閉じることを確認。  
   - 他モーダル（CSV一括・無視確認）の従来挙動に変更がないことを確認。  
   - 必要に応じてステージングでブラウザテスト。

## 5. まとめ

- **目的**: FAQ新規登録モーダルで、入力途中の誤クリック・ESC・×でモーダルが閉じず、**「保存かキャンセルをクリックしてください」**の警告を出し、データ損失を防ぐ。
- **方針**: 大原則（根本解決・シンプル・統一・具体的・安全確実・Docker必須）に沿い、**共通 Modal に `beforeClose` を追加**し、**FaqForm の下書き状態（isDirty）** と **FaqManagement の警告表示**で対応する。
- **影響範囲**: Modal のオプション追加は後方互換。FAQ追加・編集モーダルのみガードをかけ、他モーダルは変更なし。
- **次のアクション**: 指示があった時点で、上記ステップに従い実装・Docker およびステージングでの検証を実施する。
