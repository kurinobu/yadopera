## 2. Brevo フリープランでのファイル添付の可否（再確認）

### 2.1 調査結果

| 項目 | 内容 |
|------|------|
| **API仕様** | Brevo トランザクションメール API（`POST /v3/smtp/email`）は **attachment** パラメータをサポート。`content`（base64）と `name`（ファイル名）を指定可能。 |
| **対応形式** | xlsx, csv, txt, pdf, md 等、多数の形式が公式仕様に列挙されている。本件で許容する .xlsx / .csv / .txt / .md はいずれも対象内。 |
| **フリープラン** | 公式 API リファレンスには「フリープランでは添付不可」という記述は**見当たらない**。キャンペーン（マーケティング）メールの添付については「4MB/通」等の制限説明がある。トランザクションは同一 API を使用するため、実装時は 1 ファイル 10MB 以内に抑え、問題が出た場合は Brevo ダッシュボード・サポートで確認する。 |
| **結論** | **Brevo トランザクションメールでファイル添付付き送信を行う方針で進めてよい。** 送信先は Formspree ではなく、バックエンド API + Brevo のみとする。 |

## 4. 既存計画との整合性（矛盾・抜け・順序）

### 4.1 実装計画書・実行計画との照合

| 確認項目 | 結果 |
|----------|------|
| **Step 1 → 2 → 3 の順** | 計画書 §6・§7 のとおり。マニュアル → 運用開始・スクリプト → フロート申し込みの順で問題なし。 |
| **各ステップ前にバックアップ** | §7 に「各ステップ完了時に」とあるが、**着手前**にバックアップを取る運用で矛盾なし。 |

| **送信先パス** | `POST /api/v1/admin/csv-bulk-request`（apiClient の baseURL が `/api/v1` のため、`apiClient.post('/admin/csv-bulk-request', formData)` で正しい）。 |
| **フォーム項目** | 施設名・プラン・希望件数・**希望言語**（ja, en, zh-TW, zh-CN, fr, ko, es のチェック等）・連絡メール・担当者名・FAQファイル（任意）・その他要望。施設名・プラン・メール・担当者名は施設APIから自動入力可能。 |
| **503 時表示** | 「申し込み受付は一時的に利用できません。お問い合わせフォームからご連絡ください。」 |

### 4.2 抜け落ちの有無

- **実装完了の定義**: フロート申し込み機能の「実装完了」＝**コードの実装** ＋ **申し込みメールが運営に届く設定**（`ADMIN_NOTIFICATION_EMAIL` と `BREVO_API_KEY` をステージング・本番で設定すること）。未設定のままでは客が申し込んでも運営に届かず、運用不能である。
- **ADMIN_NOTIFICATION_EMAIL**: 運用開始チェックリストおよび実行計画 Phase 5 で「必須」と明記する。未設定時は API が 503 を返す。  
- **希望言語の送信形式**: 実行計画では「複数チェックボックス」とある。バックエンドでは `csv_languages` を 1 文字列（カンマ区切り等）で受け、メール本文にそのまま記載すれば足りる。  
- **マニュアル編集後**: 必読（`docs/マニュアル更新・改訂_必読.md`）に従い、フロント再ビルド・サイトデータ消去・SW解除・ハードリロードを実施する。

### 4.3 順序の確認

1. **Step 1（マニュアル）**: 4.8 の詳細化・テンプレート一元記載・Excel/スプレッドシート/Numbers 保存手順・エラー時対処。xlsx テンプレートは任意。  
2. **Step 2（有料オプション運用開始）**: スクリプト・運用開始チェックリストをバックアップから復元するか §4.2 で再作成。README に実行例を記載。  
3. **Step 3（フロート申し込み）**: バックエンド（EmailService メソッド追加 → csv_bulk_request ルーター → router 登録）→ フロント（CsvBulkRequest.vue 新規作成 → ルート・サイドバー・FaqManagement・HelpModal）。  

上記順序で矛盾なし。
