## 2. 命名規則（統一案）

### 2.1 基本フォーマット

```
Phase1_YYYYMMDD_HHMM_文書タイトル.md
```

**例**:
- `Phase1_20251205_1830_管理画面ブラウザテスト_進行状況.md`
- `Phase1_20251205_1711_引き継ぎ書.md`

### 2.2 命名規則の詳細

**構成要素**:
1. **Phase番号**: `Phase1`、`Phase2`など（統一）
2. **日時**: `YYYYMMDD_HHMM`形式（例: `20251205_1830`）
3. **文書タイトル**: アンダースコア区切り（例: `管理画面ブラウザテスト_進行状況`）

**日時フォーマット**:
- `YYYYMMDD_HHMM`: 年4桁、月2桁、日2桁、時2桁、分2桁
- 例: `20251205_1830`（2025年12月5日 18:30）

**文書タイトルの規則**:
- アンダースコア（`_`）で区切る
- 長すぎる場合は省略可能
- カテゴリを先頭に付ける（例: `ブラウザテスト_`、`引き継ぎ_`、`調査分析_`など）

### 2.3 カテゴリ別の命名例

**引き継ぎ書**:
- `Phase1_YYYYMMDD_HHMM_引き継ぎ書.md`

**ブラウザテスト関連**:
- `Phase1_YYYYMMDD_HHMM_ブラウザテスト_進行状況.md`
- `Phase1_YYYYMMDD_HHMM_ブラウザテスト_結果_完全回答.md`
- `Phase1_YYYYMMDD_HHMM_ブラウザテスト_残存課題.md`

**調査分析レポート**:
- `Phase1_YYYYMMDD_HHMM_調査分析_問題名.md`

**実装完了レポート**:
- `Phase1_YYYYMMDD_HHMM_実装完了_機能名.md`

## 3. 整理方針

### 3.1 インデックスファイルの作成

**目的**: 文書の一覧と最新版を明確にする

**ファイル名**: `Phase1_インデックス.md`

**内容**:
- 文書の一覧（カテゴリ別）
- 最新版の文書へのリンク
- 更新履歴

### 3.2 サブフォルダの整理

**提案**: カテゴリ別にサブフォルダを作成

**構造案**:
```

├── Phase1_インデックス.md
├── Phase1_YYYYMMDD_HHMM_引き継ぎ書.md
├── ブラウザテスト/
│   ├── Phase1_YYYYMMDD_HHMM_ブラウザテスト_進行状況.md
│   ├── Phase1_YYYYMMDD_HHMM_ブラウザテスト_結果_完全回答.md
│   └── ...

│   └── ...
└── 実装完了/
    ├── Phase1_YYYYMMDD_HHMM_実装完了_機能名.md
    └── ...
```

### 3.3 バックアップファイルの整理
