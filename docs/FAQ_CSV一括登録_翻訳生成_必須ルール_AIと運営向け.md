## 0. まず結論（誤解しやすい点の明文化）

### 0.1 「囲まない」のか？

**いいえ。方針は「囲む」です。**

- CSV で **列ズレを防ぐ最も確実で一般的な方法**は、**区切りと衝突しうるフィールドをダブルクォート `"..."` で囲む**ことです。
- 前回説明で「区切りと衝突しないようにする」と併記したのは、**補助手段**（金額を `1000` と書く、TSV にする等）を列挙したものであり、**「クォートしないでよい」という意味ではありません。**
- **AI が CSV を生成する場合のデフォルト**: **すべてのデータセル（ヘッダー除く）をダブルクォートで囲む（QUOTE_ALL 相当）**ことを推奨する。これにより、英語のカンマ句読点・金額表記・中国語文中の誤った半角カンマ混入にも強くなる。

### 0.2 「¥1,000」のような表記について（運営への影響）

- **半角カンマ `,` を含む文字列**（例: `¥1,000`、`5,000円`）を、**クォートなし**でセルに書くと **列が分裂**し、**中国語が途中で切れ、隣列に次の言語が流れ込む**などの重大不具合になる。
- この表記は **AI が勝手に採用したスタイル**であり、運営が Excel 上で防ぎようがない。**責任は生成側（AI・スクリプト）にある。**
- 以降、本書に従い:
  - **A案（推奨）**: 金額は **`1000円` / `1000日圓` / `¥1000`（カンマなし）** に統一する。  
  - **B案**: `¥1,000` を使う場合は **必ずそのセル全体を `"..."` で囲む**（QUOTE_ALL なら自動で安全）。

## 1. 列ズレの技術的原因（再掲）

- やどぺらの FAQ CSV は **カンマ区切り**として `pandas.read_csv` で読まれる（`csv_parser.py`）。
- **区切り文字は半角カンマ `,`（ASCII U+002C）**。
- **全角読点「，」（U+FF0C）** は区切りにならない。問題になりやすいのは主に:
  - **英語・フランス語などの文の中の半角カンマ**
  - **千の位区切りの `1,000` / `5,000`**
- 中国語「だから列ズレしやすい」の正確な言い方は、**「中国語以外でも半角カンマが入れば同じ」**。**中国語ブロックに千位区切りの半角カンマを書いた場合に顕在化しやすい**、という理解がよい。

## 2. AI・生成スクリプト向け 必須ルール（優先順）

### ルール R1 — クォート方針（最優先）

- **推奨**: CSV 出力時に **`csv.QUOTE_ALL`**（Python）または同等で、**データ行の全セルをダブルクォートで囲む。**
- 手編集で1行だけ書く場合も、**質問・回答など可変長のセルは必ず囲む。**
- セル内に `"` がある場合は **RFC 4180 どおり `""` にエスケープ**する。

### ルール R2 — 金額・数値の半角カンマ

- **原則**: FAQ 本文に **千位区切りの半角カンマを使わない**（`1000円`、`5000円`、`1000 yens` 等）。
- どうしても `1,000` 形式にする場合は **R1（クォート）を必ず満たすこと**。クォート漏れは **重大インシデント**とみなす。

### ルール R3 — 改行

- セル内改行を使う場合は **セル全体をダブルクォートで囲む**（未囲みの改行は行分割になりうる）。

### ルール R4 — 文字コード

- **UTF-8（BOM 付き推奨）** で保存（マニュアル・既存ガイドと整合）。BOM なしでも動くが、Excel 利用者向けには BOM 推奨。

### ルール R5 — 納品前の機械検証（必須）

納品前に **必ず** 次を実行し、**ヘッダーの列数と全データ行の列数が一致**することを確認する。

```bash
python3 -c "
import csv
from pathlib import Path
p = Path('納品するファイル.csv')
with p.open(newline='', encoding='utf-8-sig') as f:
    rows = list(csv.reader(f))
header_n = len(rows[0])
bad = [(i+1, len(r)) for i, r in enumerate(rows[1:]) if len(r) != header_n]
print('header columns:', header_n)
print('OK' if not bad else f'COLUMN MISMATCH rows: {bad[:20]}')
"
```

- `COLUMN MISMATCH` が1行でもあれば **納品禁止**。修正してから再検証。

### ルール R6 — やどぺらのスキーマ準拠

- **必須カラム**: `category`, `language_ja_question`, `language_ja_answer`
- **category** は次のいずれか（小文字）: `basic`, `facilities`, `location`, `trouble`
- **多言語列**（Premium 想定）:  
  `language_en_*`, `language_zh-TW_*`, `language_fr_*`, `language_ko_*`  
  カラム名は `csv_parser.py` の `LANG_PAIRS` と **完全一致**（例: `language_zh-TW_question` のハイフン）。
- **文字数**: 質問 500 文字以内、回答 2000 文字以内（超過はパーサーでエラー）。

詳細は `docs/CSV_bulk_registration_function_implementation_plan/files/FAQ_CSV_format_guide.md` を参照。

### ルール R7 — 翻訳品質（最低限）

- 日本語原文の意味（特に **金額・時間・場所・条件**）を変えない。
- 言語ごとに **誤用しやすい語**を避ける（過去事例: 朝食「1食○円」を韓国語で **「식당」(レストラン)** と訳さない → **「식사」等**）。
- 固有名詞（駅名・店名・Wi‑Fi SSID）は原文・顧客指示を優先。

### ルール R8 — 観光地・固有名詞の表記統一

- 同一行の多言語で **別の観光地を指しているように読める表現**（例: 日本語「江戸城」、フランス語のみ「皇居」寄り）を避け、**顧客確認または統一表記**とする。

## 4. 運営が AI に依頼するときの指示文

### 4.1 短い指示（コピペ用）

依頼文の先頭に付けるとよい。

```
次の制約で FAQ 用 CSV を生成・翻訳すること:
- 区切りはカンマ。列ズレ防止のため、全データセルをダブルクォートで囲む（QUOTE_ALL 相当）。
- 金額は半角カンマを使わない（例: 1000円、¥1000）。どうしても 1,000 形式にする場合も必ずクォート内に含める。
- 必須カラム category, language_ja_question, language_ja_answer。category は basic|facilities|location|trouble のみ。
- UTF-8 BOM 付きで出力する想定で内容を生成。
- 納品前に全行の列数がヘッダーと一致することを自己検証する手順を説明に含める。
- やどぺら backend の csv_parser.py の LANG_PAIRS に合わせた列名を使う。
```

### 4.2 具体的な依頼テンプレート（長文・状況に合わせて編集）

運営が Cursor 等の AI に「CSV を作って／翻訳して」と依頼するときは、**次のブロックをコピーし、括弧内を埋めてから送る**と抜け漏れが減る。

```
【前提】
- やどぺらの FAQ CSV 一括登録向け。必ずリポジトリ内の次を読んだうえで作業すること:
  - docs/FAQ_CSV一括登録_翻訳生成_必須ルール_AIと運営向け.md（特に §0〜§2）
  - backend/app/services/csv_parser.py の LANG_PAIRS（列名は完全一致）

【入力素材】
- （例: 添付の日本語原稿 / 既存CSVのパス / 施設名とプラン）

【やってほしいこと】
1. （例: テンプレートに沿った CSV を新規作成する / 既存CSVに韓国語列を追加し翻訳を埋める）
2. 出力は UTF-8（BOM 付き想定）。区切りは半角カンマ。列ズレ防止のためデータ行は QUOTE_ALL 相当（全セルを "..." で囲む）。
3. 金額・数値は千位の半角カンマを使わない（例: 1000円）。どうしてもカンマを含める場合は必ずクォート内。
4. 必須列: category, language_ja_question, language_ja_answer。
   任意列: intent_key（通常は空欄でよい）, priority（省略時はシステム既定）, is_active（true/false、省略時 true）。
5. 多言語列名は次のとおり（大文字小文字・ハイフン含め完全一致。誤り例: language_zhtw_question は NG）:
   - 英語: language_en_question / language_en_answer
   - 繁体中国語: language_zh-TW_question / language_zh-TW_answer（zh と TW の間はハイフン必須）
   - フランス語: language_fr_question / language_fr_answer
   - 韓国語: language_ko_question / language_ko_answer
   ※ ゲスト画面で簡体中国語・スペイン語などが選べても、CSV 一括のパーサーが読むのは上記＋日本語まで。zh-CN / es 用の列を足しても一括アップロードでは取り込まれない（別途管理画面で登録が必要）。
6. is_active: true = ゲスト向けに有効（FAQ 検索・AI 参照の対象になりうる）。false = 無効（事実上ゲストには出さない想定で登録する行に使う）。
7. 納品前に §2 R5 の Python ワンライナーで「全行の列数＝ヘッダー列数」を確認し、不一致があれば修正してからファイルを提示する。

【納品物】
- ファイル名: （例: FAQ_施設名_YYYYMMDD.csv）
- 変更内容の要約: （言語追加・件数・注意した表記 など）
```
