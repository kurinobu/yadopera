## 1. 目的・全体像の整理

### 1.1 本タスクの位置づけ

| 項目 | 内容 |
|------|------|
| **目的** | LP の主導線を邪魔せず、「調べたい人向け」の補助情報をまとめた**静的ページ群**を整備する。 |
| **4a** | **トップ（思想＋ミッション）** — 1ページ。YadOPERA の考え方・目指すものを伝える。 |
| **4b** | **運営情報／セキュリティ方針／導入事例アーカイブ／補助金・制度解説／メディア・お知らせ** — 各1ページ（または既存ページへの統合）。 |

### 1.2 本番LP との関係（重要）

本番向けLP計画書および LP_v1.0 で定義されている関係は以下のとおりです。

```
                    yadopera.com（GitHub Pages）
                              │
        ┌─────────────────────┼─────────────────────┐
        │                     │                     │
        ▼                     ▼                     ▼
   / (index.html)      /about.html            /legal/*
   本番LP（申し込み    ホームページ最小版      既存リーガル
   に特化した導線）    トップ（思想・ミッション） 運営・プライバシー等
        │                     │                     │
        │ ヘッダー「補助金・  │                     │
        │ 制度対応について」──┼─────────────────────┼──→ /subsidy.html
        │ フッター「運営」等 ─┼─────────────────────┼──→ 既存 legal/* へ
        │                     │                     │
        └─────────────────────┴─────────────────────┘
```

| 役割 | 本番LP（順序2） | ホームページ最小版（本タスク・順序4） |
|------|------------------|----------------------------------------|
| **役割** | 申し込みに特化した1本の導線。yadopera.com の**入口**＝LP。 | 補助情報のまとまり。LP からリンクされる「調べたい人向け」の別ページ群。 |
| **内容** | ヒーロー、課題、解決策、料金、導入の声、CTA（Freeプランで今すぐ〜）。 | トップ（思想＋ミッション）、運営情報／セキュリティ方針／導入事例アーカイブ／補助金・制度解説／メディア・お知らせ。 |
| **実施順・依存** | 先に実施済み（develop マージ済み。本番デプロイは別指示）。 | **本タスク**。LP のヘッダー「補助金・制度対応について」のリンク先（現在 `#`）を、本タスクで作成するページに差し替える。 |

**結論**:  
- **LP** ＝ ドメインの入口であり「売る・申し込ませる」ための単一ページ。  
- **ホームページ最小版** ＝ LP からリンクされる**補助ページ群**（思想・ミッション＋運営・セキュリティ・事例・補助金・メディア等）を整備するタスク。  
- 構造上は **LP が親（入口）、ホームページ最小版が子（補助情報）**。

### 2.3 大原則との整合（プロジェクト現況総括 §0.3）

| 原則 | 本タスクでの反映 |
|------|------------------|
| 根本解決 > 暫定解決 | リンク先を `#` のままにせず、実ページを用意してヘッダー・フッターから正しくリンクする。 |
| シンプル構造 > 複雑構造 | 静的 HTML のまま。Vue に組み込まず、`landing/` 配下に静的ファイルを追加するのみ。 |

| Docker 環境必須 | LP/静的サイトは Docker 不要。変更後の確認はローカル表示または GitHub Pages プレビューで足りる。 |
| Vercel 使用厳禁 | 使用しない。配信は **GitHub Pages のみ**。 |

## 4. 各ページの内容方針（4a・4b）

### 4a. トップ：思想＋ミッション（1ページ） — `about.html`

| 項目 | 内容 |
|------|------|
| **目的** | 「YadOPERA とは何か」「なぜこのサービスがあるか」を、申し込み導線ではなく**説明用**に1ページで伝える。 |
| **想定読者** | 導入を検討する施設担当者、メディア・パートナー、補助金申請で「事業内容」を説明する担当者。 |
| **構成案** | 見出し例：ミッション（説明を止めない／一次対応／判断は人に戻す）、やらないこと（医療・安全は即スタッフ等）、ターゲット（ゲストハウス・ホステル・小規模ホテルなど、スタッフ1〜20名規模等）。LP 本文と重複しないよう、**思想・背景・ミッション**に絞る。 |
| **CTA** | 「申し込む」は LP に任せる。about ページ末尾に「Freeプランで始める」→ `index.html` または LP の CTA URL へのリンクを1本置く程度で可。 |

**about.html の見出し案（イメージ）**  
- H1: 私たちについて / YadOPERA の思想とミッション  
- ミッション：説明を止めない・一次対応・判断は人に戻す  
- やらないこと：医療・安全は即スタッフ、無理な自動化をしない  
- こんな施設を想定しています：ゲストハウス・ホステル・小規模ホテルなど、スタッフ1〜20名規模  
- お問い合わせ・Freeプランで始める（LP へのリンク）

### 4b-1. 運営情報 — 既存 `legal/operation.html` を利用

- 追加実装は不要。フッター・ヘッダーから「運営」として既にリンク済み。ホームページ最小版の「運営情報」はこのページで賄う。

### 4b-2. セキュリティ方針 — `legal/security.html`（新規）

| 項目 | 内容 |
|------|------|
| **目的** | セキュリティ・データ保護の方針を明文化し、信頼性を補強する。 |
| **内容案** | 要約定義書・プライバシーポリシーを踏まえ、以下のような項目を短文で記載：通信の暗号化（SSL/TLS）、認証・パスワードの取扱い、個人情報の保存期間・委託先、アクセス制限、インシデント時の考え方。プライバシーポリシーと重複しすぎないよう「方針」レベルに留める。 |

### 4b-3. 導入事例アーカイブ — `cases.html`（新規）

| 項目 | 内容 |
|------|------|
| **目的** | LP_v1.0 の「導入事例・導入店一覧」に相当。社会的証明・補助金申請時の実績活用。 |
| **本番ローンチ時点** | LP_v1.0 では「運用実績ができてから 2・3 を掲載」とある。**初回リリース時は「準備中」「順次掲載予定」など短い文言で1ページ用意**し、後から事例を追記する形でよい。 |
| **将来** | Free／有料プラン導入施設一覧、地域・業態・規模、コメント（任意）を掲載。 |

### 4b-4. 補助金・制度解説 — `subsidy.html`（新規）

| 項目 | 内容 |
|------|------|
| **目的** | LP_v1.0「1. 補助金・制度対応について」に対応。経営者・法人担当者向けの判断材料。 |
| **内容** | LP_v1.0 に従い：デジタル化・AI導入支援制度の概要、YadOPERA が制度対象となりうる理由、対象となりうるケース、申請時の一般的な注意点。**断定表現は避ける**。要約定義書の補助金戦略（小規模事業者持続化補助金、IT導入補助金等）を参照しつつ、申請は利用者自身の判断であることを明記。 |

**subsidy.html の見出し案（イメージ）**  
- H1: 補助金・制度対応について  
- はじめに（本ページの目的、申請はご自身の判断である旨）  
- 関連しうる制度の例（小規模事業者持続化補助金、IT導入補助金等の概要のみ。対象かどうかは各制度の要件をご確認ください、の文言）  
- YadOPERA が対象となりうる理由（デジタル化・AI導入・業務効率化の文脈で参照されうる、程度の表現）  
- 申請時の注意点（一般論：申請期限・要件確認・書類はご自身で準備等）  
- お問い合わせ（support@yadopera.com 等）

### 4b-5. メディア・お知らせ — `news.html`（新規）

| 項目 | 内容 |
|------|------|
| **目的** | プレスリリース・メディア掲載・サービスお知らせを一覧する。 |
| **初回** | 「お知らせは順次掲載します」等の短文でよい。日付・タイトル・リンク（または本文要約）を並べる形式にしておき、後から追記しやすくする。 |
