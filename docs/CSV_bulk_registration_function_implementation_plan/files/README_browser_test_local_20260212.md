# ローカルブラウザテスト チェックリスト（2026-02-12 修正後）

**※ 問題A・B 解決後の「即座に実施」用の完全版は同フォルダの `README_ローカルブラウザテスト_問題AB解決後_20260212.md` を参照してください。** 以下は従来版のチェックリストです。

**前提**: フロントは `http://localhost:5173`、バックエンドは `http://localhost:8000` で起動していること。未ログインの場合は管理画面用アカウントでログインする。

---

## 確認項目（3件の修正に対応）

### 1. CSV が配信される（問題C 対応）

- [ ] ブラウザで **http://localhost:5173/faq-csv-template/FAQ_CSV_template_4lang.csv** を直接開く
- [ ] **404 にならず**、CSV の中身（例: 1行目が `category,intent_key,...`）が表示される、またはファイルがダウンロードされる

※ 開発サーバー（`npm run dev`）では `public/` がルートで配信されるため、上記 URL でアクセス可能。本項目は主に本番ビルド用の確認だが、ローカルでも URL の挙動確認として実施可。

---

### 2. マニュアルの「CSVテンプレートをダウンロード」がリンクになる（問題A 対応）

- [ ] **http://localhost:5173/admin/manual** を開く
- [ ] 4.8「CSV一括登録」までスクロールし、「マニュアルのこの節からも CSVテンプレートをダウンロード できます」の **「CSVテンプレートをダウンロード」が青リンク**になっている
- [ ] そのリンクをクリックすると、CSV が開くまたはダウンロードされる
- [ ] **いったん別ページ（例: FAQ管理）に移動してから、再度 マニュアル に戻る**
- [ ] 再度 4.8 を表示し、**「CSVテンプレートをダウンロード」が再表示後も青リンクのまま**で、コード（`[CSVテンプレートをダウンロード](...)`）がそのまま表示されていない

---

### 3. モーダルで「フォーマットのダウンロード」が表示される（問題B 対応）

- [ ] **http://localhost:5173/admin/faqs** を開く
- [ ] **「CSV一括登録」** ボタンをクリックしてモーダルを開く
- [ ] モーダル内に **「推奨テンプレート（…）をダウンロードして編集し…」** の説明と **「CSVテンプレートをダウンロード」** リンクが表示されている
- [ ] モーダルを **閉じる**
- [ ] 再度 **「CSV一括登録」** ボタンをクリックしてモーダルを開く
- [ ] **2回目も** 上記のテンプレート説明と「CSVテンプレートをダウンロード」リンクが表示されている

---

## 起動コマンド（参照）

```bash
# プロジェクトルートで
docker compose up -d

# フロントのみ（バックエンドは別途起動している場合）
cd frontend && npm run dev
```

- フロント: http://localhost:5173
- バックエンド: http://localhost:8000
- API ドキュメント: http://localhost:8000/docs
