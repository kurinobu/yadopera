## 1. ご利用マニュアル（Manual.vue）の現況分析

### 1.1 マニュアルの構成

**全体構造**:
- 第1章 はじめに（4項目）
- 第2章 ログイン・ログアウト（3項目）
- 第3章 ダッシュボードの見方（7項目）
- 第4章 FAQ管理（7項目）
- 第5章 スタッフ不在時間帯対応キュー（3項目以上）
- 第6章～第12章（省略部分あり）

**合計**: 12章構成、各章2～7の項目で構成

### 1.2 文体・表現の特徴

#### **敬語レベル**
- ✅ **丁寧体（です・ます調）で統一**
  - 例: 「表示されます」「確認してください」「アクセスできます」
  - 「です」「ます」を多用
  - 全文で統一されている

#### **ユーザー向けの親切な表現**
- ✅ **二人称は「あなた」ではなく直接的な指示**
  - 例: 「メールアドレスを入力（例: `admin@example.com`）」
  - 「パスワードを入力（8文字以上）」
  - 「クリック」「タップ」などのUI操作用語を明確に

- ✅ **括弧で補足説明を追加**
  - 例: 「（例: 22時以降は...）」
  - 「（詳細は「第10章 トラブルシューティング」の「10.3 ブラウザキャッシュのクリア方法」を参照してください）」
  - 詳細箇所への導線を丁寧に記載

#### **視覚的な強調手法**
- ✅ **絵文字は使用しない** （ただし、「💡ポイント」「⚠️注意」「✅ベストプラクティス」は使用可能）
- ✅ **見出し記号の使用**
  - `**太字**` で強調
  - `#`, `##`, `###` で階層構造を表現
  - `-` で箇条書き
  - 数字付き順序リスト

#### **セクション構成**
- ✅ **各セクションの構成パターン**:
  ```
  ## セクションタイトル
  
  **説明1**:
  - 定義や説明文
  
  **説明2**:
  - 詳細な内容
  
  **説明3**:
  - さらに詳細な内容
  ```

- ✅ **「～について」「～の見方」「～の手順」「～の注意点」「～の活用方法」パターンで統一**

#### **会話的要素**
- ✅ **直接的でわかりやすい説明**
  - 「～で、～が表示されます」
  - 「～をすると、～に遷移します」
  - 「～のため、～です」

### 1.3 具体例分析

#### **例1: 基本情報セクション（第1章1.2節）**
```markdown
**本マニュアルの使い方**:
- 左側の目次から章・項目をクリックすると、該当箇所にスクロールします
- 現在表示中の章は青くハイライトされます
- 章タイトルをクリックすると、その章の項目が折りたたみ/展開されます
```

**特徴**:
- 丁寧体・直接的指示を統一
- 括弧で補足説明
- 箇条書きで明確に
- 「～すると、～します」パターン

#### **例2: 手順説明（第2章2.1節）**
```markdown
**ログイン手順**:
1. メールアドレスを入力（例: `admin@example.com`）
2. パスワードを入力（8文字以上）
3. 「ログイン / Login」ボタンをクリック
```

**特徴**:
- 番号付き順序リスト
- 括弧で条件・例を補足
- 英語と日本語の両記載（UI的に両言語対応）
- 「～を入力」「～をクリック」で一貫性

#### **例3: 注意事項（第4章4.5節）**
```markdown
**FAQ削除時の注意点**:
- 削除したFAQは復元できません
- 削除後、そのFAQはAI回答から参照されなくなります
- 削除前に、そのFAQが本当に不要かどうか確認してください
- 一時的に無効化したい場合は、削除ではなく「有効/無効切替」を使用してください
```

**特徴**:
- ネガティブな内容（注意）も丁寧に説明
- 代替案を提示
- 「～前に、～してください」パターン

## 3. 文体・表現の整合性分析

### 3.1 相違点（修正が必要な項目）

| 項目 | マニュアル | 翻訳手引書 | 判定 |
|------|-----------|---------|------|
| 敬語レベル | 丁寧体（です・ます）統一 | 丁寧体（です・ます）統一 | ✅ 一致 |
| 二人称表現 | 直接的指示（「～してください」） | 直接的指示（「ください」） | ✅ 一致 |
| 絵文字使用 | 使用しない（通常） | ⚠️ 使用（✅、❌、💡、⚠️等） | ⚠️ **相違** |
| 強調手法 | `**太字**`、括弧補足 | 見出し、コードブロック、絵文字 | ⚠️ **部分相違** |
| セクション構成 | 「～について」「～の見方」「～の手順」パターン | 「～とは」「基本手順」「コツ」パターン | ⚠️ **異なる** |
| 補足説明 | 括弧で補足 | コードブロックで詳細 | ⚠️ **異なる** |

### 3.2 相違点の詳細

#### **相違1: 絵文字の使用**

**マニュアル側**:
```markdown
💡 **ポイント**: 重要な情報やヒント
⚠️ **注意**: 注意が必要な事項
✅ **ベストプラクティス**: 推奨される使い方
```
※ ただしコンテンツには絵文字の記載があるが、実装（script部分）では使用していない

**翻訳手引書側**:
```markdown
**特徴**（✅ 評価良好）:
- ✅ 迷わず安全に高品質な翻訳ができる設計

**課題**（⚠️ 改善余地あり）:
- ⚠️ 手引書は「宿泊施設管理者向けの翻訳ガイド」だが...
```

**判定**: 
- マニュアル: テンプレート記載あるが、実装時に使用有無は不明
- 翻訳手引書: アクティブに絵文字を使用

**推奨**: **統一する場合は、マニュアル側のテンプレート（💡⚠️✅）に合わせることが自然**

#### **相違2: セクション構成パターン**

**マニュアル**:
```markdown
## 2.1 初回ログイン

**ログイン画面へのアクセス**:
- ～

**ログイン手順**:
1. ～
2. ～

**ログイン成功後**:
- ～
```
→ 「～について」「～の見方」「～の手順」「～の注意点」で統一

**翻訳手引書**:
```markdown
## 3. ChatGPTで翻訳する基本手順（初心者向け）

### Step 1：ChatGPTを開く
- ～

### Step 2：翻訳用の指示文（コピペ用）
```
→ 「Step 1」「Step 2」など段階的な「手順」パターン

**判定**: 相互に相補的（マニュアルは「何か」の説明、翻訳手引書は「どうするか」の手順）

**推奨**: **付録として掲載する場合は、翻訳手引書のセクション番号を「付録A」「付録B」等に変更し、マニュアルの12章体制に統合する**

#### **相違3: 補足説明の方法**

**マニュアル**:
```markdown
**ログイン手順**:
1. メールアドレスを入力（例: `admin@example.com`）
2. パスワードを入力（8文字以上）
```
→ 括弧で簡潔に補足

**翻訳手引書**:
````markdown
```
あなたは宿泊業界専門の翻訳者です。
以下の日本語FAQを、自然で正確な英語と繁体中国語に翻訳してください。
（コピペ可能なテンプレート）
```
````
→ コードブロック（```）で大きなテンプレートを示す

**判定**: 用途が異なるため、両方を活用すべき

**推奨**: **翻訳手引書の「コピペ用テンプレート」はそのまま活用し、簡単な補足はマニュアル側のパターン（括弧）に統一する**

**選択肢**:
- **案1（階層化）**: より詳細だが、スクロール量が増加
- **案2（統合）**: よりシンプルだが、セクション内での見出し操作が必要

### 6.2 目次（ManualToc.vue）への影響

**現状**: 自動生成される可能性が高い（sections配列から自動生成）

**統合後の動作**:
- ✅ subsections に新規セクションが追加されれば、目次に自動反映
- ✅ 階層化した場合、「12.2 翻訳手引書」→「12.2.1 翻訳手引書について」の構造が反映
- ⚠️ 目次の階層が深くなる可能性（3階層に）

### 6.3 SearchおよびIntersectionObserver への影響

**IntersectionObserver**:
- ✅ `id`属性が正しく設定されていれば、スクロール位置検出に影響なし
- ⚠️ 新規セクションが追加される場合、`id`の一意性を確認

**Search機能** （存在する場合）:
- ✅ テキスト内容のみで検索されるため、セクション構造の変更は影響なし
