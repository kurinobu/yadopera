## 0. 目的・スコープ・大原則

### 0.1 目的

- ゲストが選択した言語（`route.query.lang`）に応じて、**ウェルカム画面のUIラベル・見出し・緊急連絡先**を表示する。
- 緊急連絡先（救急・消防・警察・施設連絡先）は**緊急時に誰でも意味が分かる**よう、選択言語で表示する（根本解決）。

### 0.2 スコープ

| 対象 | 内容 |
|------|------|
| **第一次** | ウェルカム画面: 緊急連絡先、よくある質問見出し、メッセージ入力ラベル・プレースホルダ、施設ヘッダー（Check-in/Check-out/Loading） |
| **第二次（任意）** | チャット画面の「読み込み中...」「AIが回答を生成中...」等。本計画では第一次完了を必須とする。 |

### 0.3 大原則との対応（総括 §0.1）

| 大原則 | 本件での適用 |
|--------|----------------|
| **根本解決 > 暫定解決** | ラベルをゲストの選択言語で一貫表示。緊急連絡先は緊急時でも理解できるようにする。 |
| **シンプル構造 > 複雑構造** | 1 モジュール（`guestCopy.ts`）にゲスト用文言を集約し、既存の `couponCopy.ts` と同じパターンで扱う。 |
| **統一・同一化 > 特殊独自** | `couponCopy.ts` と同様の「言語コード → 表示文言」＋ `getGuestCopy(lang)` で統一。緊急連絡先専用モジュールは作らず `guestCopy` に統合。 |
| **拙速 < 安全確実** | 対応言語は `SUPPORTED_LANGUAGES` と揃え（8言語）、未対応は en にフォールバック。Docker でビルド・動作確認後にデプロイ。 |
| **Docker環境必須** | すべてのビルド・テストは Docker 上で実行する。 |

## 1. 現況サマリ（コードベース調査）

### 1.1 ゲスト画面の言語まわり

- **言語の決定**: `Welcome.vue` で `language = computed(() => (route.query.lang as string) || 'en')`。API には `facilityApi.getFacility(slug, location, language)` で渡しており、FAQ 本文は選択言語で返る。
- **言語カード**: `LanguageSelect.vue` が `SUPPORTED_LANGUAGES` を `available_languages` でフィルタし、表示順は `constants.ts` の定義順。API の `available_languages` は「表示する言語の集合」の正（総括・ステージング検証チェックリストに準拠）。

### 1.2 固定文言になっている箇所

| コンポーネント | ファイル | 固定文言（現状） |
|----------------|----------|------------------|
| EmergencyContact | `EmergencyContact.vue` | 「緊急連絡先 / Emergency Contact」「救急・消防」「警察」「施設連絡先」 |
| TopQuestions | `TopQuestions.vue` | 「よくある質問 / Frequently Asked Questions」「よくある質問はありません」 |
| MessageInput | `MessageInput.vue` | 「メッセージを入力 / Enter your message」「質問を入力してください...」 |
| FacilityHeader | `FacilityHeader.vue` | 「Check-in:」「Check-out:」「Loading...」 |

### 1.3 既存パターン（couponCopy.ts）

- `frontend/src/utils/couponCopy.ts`: 8言語（ja, en, zh-TW, zh-CN, fr, es, ko, th）のオブジェクトを `COPY_MAP` で保持し、`getCouponCopy(lang: string)` で返す。未対応は `en` にフォールバック。
- ゲスト用文言も同じ方式で 1 ファイルにまとめ、`getGuestCopy(lang)` を用意する（統一・同一化）。

### 1.4 Docker・ブランチ

- **Docker**: `docker-compose.yml` で frontend は `npm run dev`。本実装はフロントのみ。再ビルドは `docker compose up -d --build frontend`（マニュアル必読に準拠）。
- **ブランチ**: ステージングは `develop` を基準。Render が `develop` を自動デプロイ。作業は `feature/*` で行い、完了後に `develop` へマージする。

## 4. ゲスト画面用URL（テスト用施設・ローカル）

### 4.1 調査結果

- **URL形式**: ゲスト画面のパスは `/f/{facilityId}/welcome`（および `/f/{facilityId}`, `/f/{facilityId}/chat`）。ここで `{facilityId}` は**施設 slug または数値 ID** のいずれか。
- **バックエンド**: `FacilityService.get_facility_public_info(db, slug, language)` は、第一引数に **slug** を渡す。slug で見つからない場合は **数値として解釈し ID で検索**する（`facility_service.py` 65–84 行付近）。
- **フロント**: `Welcome.vue` は `route.params.facilityId` をそのまま API の slug 引数として使用している。
- **ローカル**: ローカル環境の DB に登録されている施設は**すべてテスト用**として利用してよい。

### 4.2 テスト用施設の抽出方法

施設一覧（id・slug・名前・**プラン**）とゲスト画面用URL例を取得するスクリプトを用意した。**多言語表示の確認は Premium プラン施設で行うこと**（Free は日本語のみ、Premium は 7 言語選択可能）。

```bash
# Docker 利用時（backend の .env / コンテナの DATABASE_URL で DB に接続）
docker compose exec backend python scripts/list_facilities_for_guest_url.py
```

- **出力**: 有効な施設の ID / プラン / slug / 名前。**Premium プラン施設を先頭に表示**し、多言語テスト推奨URL例を出したあと、全施設のURL例を表示する。
- **フロントベースURL**: デフォルトは `http://localhost:5173`。ステージングで実行する場合は `FRONTEND_BASE_URL=https://yadopera-frontend-staging.onrender.com` を指定可能。

### 4.3 ゲスト画面用URL例（ローカル・Premium プラン推奨）

**多言語テストは Premium プラン施設を使用すること。** Free プランは日本語のみ表示のため、言語カードが1つしか出ない。

| 用途 | URL例（Premium 施設） |
|------|------------------------|
| 言語選択 | `http://localhost:5173/f/test-hotel-premium-d4bdfcae` |
| ウェルカム（日本語） | `http://localhost:5173/f/test-hotel-premium-d4bdfcae/welcome?lang=ja` |
| ウェルカム（英語） | `http://localhost:5173/f/test-hotel-premium-d4bdfcae/welcome?lang=en` |
| ウェルカム（フランス語） | `http://localhost:5173/f/test-hotel-premium-d4bdfcae/welcome?lang=fr` |
| ウェルカム（中国語簡体） | `http://localhost:5173/f/test-hotel-premium-d4bdfcae/welcome?lang=zh-CN` |
| ウェルカム（韓国語） | `http://localhost:5173/f/test-hotel-premium-d4bdfcae/welcome?lang=ko` |
| チャット | `http://localhost:5173/f/test-hotel-premium-d4bdfcae/chat?lang=ja` |

- **上記の slug**: `test-hotel-premium-d4bdfcae`（施設名: Test Hotel Premium, id=44）。**Premium プラン**のため、言語選択で 7 言語（ja, en, zh-TW, zh-CN, fr, ko, es）が表示される。
- 他の Premium 施設: 4.2 のスクリプト実行で「【Premium プラン施設（多言語テスト推奨）】」に一覧が出る（例: `test-facility-premiu-9455eb18`, `testhotel-7ee91d05`, `premiumホテル-b1e4c480`）。
- 数値 ID のみ使う場合: `http://localhost:5173/f/44/welcome?lang=ja`（id=44 は上記 Test Hotel Premium）。

### 4.4 施設が1件もない場合（ローカル）

- `create_test_data.py` を実行すると、slug=`test-facility` の施設が作成され、完了時にゲスト画面URLが表示される。
- 実行例（Docker）: `docker compose exec backend python create_test_data.py`（backend の作業ディレクトリ・DATABASE_URL に依存）。
