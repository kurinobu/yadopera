# マニュアル・ヘルプチャット未反映 — 原因調査と修正案

**作成日**: 2026年3月14日  
**目的**: ローカルブラウザテストで発見された「マニュアルが反映されていない」「ヘルプチャットが反映されていない」の**原因を完全に調査**し、**大原則に準拠した修正案**を提示する。  
**実施結果（マニュアル）**: 2026-03-14 に編集後必須手順（フロント再ビルド・キャッシュ消去・SW解除・ハードリロード）の実施およびデプロイにより、マニュアル 4.8（有料代行・料金の目安含む）の反映をステージングで確認完了。ヘルプチャットのCSV・代行まわりFAQ追加は別途未実施。

---

## 0. 大原則（参照）

- 根本解決 > 暫定解決  
- シンプル構造 > 複雑構造  
- 統一・同一化 > 特殊独自  
- 具体的 > 一般  
- 拙速 < 安全確実  
- Docker環境必須

## 1. マニュアルが反映されていない — 原因調査

### 1.1 事象

- 管理画面の「ご利用マニュアル」を開いても、**Step 1 で追記した 4.8「CSV一括登録（Standard・Premium）」の詳細**（手順5ステップ・エラー時対処・テンプレート一元記載・Excel/スプレッドシート/Numbers の保存方法など）が表示されていない、または古い内容のままである。

### 1.2 コンテンツの所在と現状

| 項目 | 内容 |
|------|------|
| **定義元** | `frontend/src/views/admin/Manual.vue` の `sections` 配列内、id `faq-csv-bulk` の `content`（テンプレートリテラル）。 |
| **表示担当** | `frontend/src/components/admin/ManualContent.vue` が `formatContent()` で本文を HTML に変換して表示。 |
| **現状** | **Manual.vue の 4.8 節には、Step 1 で追加した詳細コンテンツがすでに含まれている**（手順・CSV形式・テンプレート入手・エラー時対処・Excel等の保存方法まで記載済み）。つまり**コード上の「正」は更新済み**。 |

### 1.3 原因の特定

**コードは正しい。表示が変わらない原因は「ブラウザが古いアセットを参照している」ためである。**

| 要因 | 説明 |
|------|------|
| **1. PWA Service Worker（Workbox）** | `frontend/vite.config.ts` で VitePWA が有効。`workbox.globPatterns: ['**/*.{js,css,html,ico,png,svg}']` により、**一度キャッシュされた JS/CSS/HTML が Service Worker から返り続ける**。マニュアル本文は Vue コンポーネント（Manual.vue）に含まれるため、**バンドルされた JS が古いと表示も古いまま**になる。 |
| **2. ブラウザキャッシュ** | 開発時でも、ブラウザが JS/CSS をディスク/メモリにキャッシュする。リロードだけでは古いファイルが使われることがある。 |
| **3. 編集後必須手順の未実施** | `docs/マニュアル更新・改訂_必読.md` および `docs/マニュアルManual編集後の反映手順.md` に、**「フロント再ビルド・サイトデータ消去・Service Worker 解除・ハードリロード」** が必須と明記されている。これらを実施しないと、**コードを変えただけでは表示は変わらない** と同ドキュメントにも書かれている。 |

**Docker 運用時の補足**: `docker-compose.yml` では frontend は `volumes: - ./frontend:/app` で **`npm run dev`** を実行している。つまり **ビルド成果物を配信しているのではなく、Vite 開発サーバーがホストの `frontend/` をそのまま提供している**。したがって「Docker イメージが古い」のではなく、**ブラウザ側が古い JS をキャッシュまたは Service Worker 経由で取得している**ことが原因である。

### 1.4 結論（マニュアル）

- **根本原因**: マニュアル編集後の**必須手順（サイトデータ消去・Service Worker 解除・ハードリロード）が実施されていない**こと。その結果、PWA の precache とブラウザキャッシュにより、**古いバンドルが表示され続けている**。
- **コード側の不具合ではない**。Manual.vue の 4.8 は更新済み。

## 2. ヘルプチャットが反映されていない — 原因調査

### 2.1 事象

- ヘルプ（？ボタン）のチャット／FAQ で、**CSV一括登録や有料代行申し込み**に関する説明が期待どおりに出てこない、または古い・不十分な内容である。

### 2.2 ヘルプチャットの仕組み

| 項目 | 内容 |
|------|------|
| **データソース** | **DB の `operator_faqs` と `operator_faq_translations`**。フロントは API `GET /api/v1/help/faqs` で取得し、AI チャットは `OperatorHelpChatService` が**同 FAQ 全文をシステムプロンプトに埋め込んで**回答する。 |
| **データの正** | **`backend/scripts/insert_operator_faqs.py`** の `OPERATOR_FAQ_DATA`。定義書: `docs/help_system_faq_data.md`。 |
| **DB 反映** | 初回またはスキーマ一致時は `insert_operator_faqs.py`。既存レコードの更新は **`update_operator_faqs.py`**。 |
| **キャッシュ** | `OperatorFaqService` が **Redis** で 5 分 TTL（キー例: `operator_faqs:ja:all:True`）。FAQ を変更した場合、**キャッシュ削除または backend 再起動**をしないと古い内容が返る。 |

### 2.3 現状の insert_operator_faqs.py

| intent_key | 内容 | related_url |
|------------|------|-------------|
| **faq_bulk_import** | 「FAQを一括登録できますか？」→ Standard・Premium で CSV 一括登録が利用できる旨。 | **None**（申し込みページや FAQ 管理への誘導が無い） |

**存在しない項目**（ロールバック前のバックアップには存在）:

| intent_key | 内容 |
|------------|------|
| **faq_bulk_import_paid_option** | 「CSV一括登録の有料代行はありますか？どう申し込めますか？」→ 代行の案内・申し込み導線（「代行をご希望の方はこちら」・サイドバー「CSV代行の申し込み」・専用フォーム送信）の説明。 |

`docs/20260313_CSV一括登録_ロールバック先_調査報告.md` によれば、**fdd69d3 に戻す対象**に `insert_operator_faqs.py`（Step 3 関連で faq_bulk_import_paid_option 追加等）が含まれていた。つまり **現在の作業ツリーの insert_operator_faqs.py は、fdd69d3 相当（またはそれに近い）状態で、faq_bulk_import_paid_option が無い**。バックアップ `backups/20260313_before_rollback/backend/scripts/insert_operator_faqs.py` には **faq_bulk_import_paid_option** が含まれている。

### 2.4 原因の特定

| 要因 | 説明 |
|------|------|
| **1. ヘルプFAQに「CSV一括登録の有料代行」項目が無い** | **faq_bulk_import_paid_option** が現在の `OPERATOR_FAQ_DATA` に存在しない。そのため「代行はありますか？どう申し込めますか？」に答える公式の FAQ がチャットに含まれず、反映されていないように見える。 |
| **2. faq_bulk_import の related_url が未設定** | `related_url: None` のため、一括登録・申し込みページ（`/admin/faqs` や `/admin/csv-bulk-request`）への誘導がチャット回答に含まれにくい。 |
| **3. DB 未反映** | `insert_operator_faqs.py` や `help_system_faq_data.md` を更新しても、**ローカル（または該当環境）の DB で `update_operator_faqs.py`（または insert）を実行していない**と、API が返す FAQ は古いまま。 |
| **4. Redis キャッシュ** | DB を更新しても、**Redis の `operator_faqs:*` が残っている**と、最大 5 分間は古い FAQ が返る。backend 再起動またはキャッシュ削除で解消。 |

### 2.5 結論（ヘルプチャット）

- **根本原因**:  
  (1) **ヘルプ用 FAQ データ**に「CSV一括登録の有料代行」を説明する **faq_bulk_import_paid_option** が**無い**（またはロールバックで削除されたまま復元されていない）。  
  (2) **faq_bulk_import** の **related_url** が未設定で、申し込み・FAQ 管理への誘導が弱い。  
  (3) FAQ を更新した場合の **DB 反映・キャッシュ削除**が実施されていない可能性がある。

## 3. 大原則に準拠した修正案

### 3.1 マニュアルについて（根本解決・安全確実）

- **原因は運用手順の未実施**であり、コード修正は不要である。  
- **根本解決**: マニュアルを編集したあと、**必ず** `docs/マニュアル更新・改訂_必読.md` の「5. 編集後の必須手順」を実施する。  
- **修正案（手順の徹底）**:
  1. **フロントの再ビルド・再起動**  
     - Docker: `docker compose up -d --build frontend`（開発モードでも再起動によりホストの最新ソースが読まれる）。  
     - キャッシュを疑う場合: `docker compose build --no-cache frontend` のあと `docker compose up -d frontend`。
  2. **ブラウザでキャッシュと Service Worker の解除**  
     - 開発者ツール → Application → Storage → **「サイトデータを消去」**。  
     - Application → Service Workers → 登録されている SW を **Unregister**。
  3. **ハードリロード**  
     - macOS: `Cmd + Shift + R`。Windows/Linux: `Ctrl + Shift + R`。またはタブを閉じて `http://localhost:5173/admin/manual` を開き直す。
  4. **確認**  
     - 4.8 節に「使い方（手順）」5ステップ・「エラー時の対処（一覧）」・「テンプレートの入手（一元記載）」・Excel/スプレッドシート/Numbers の保存方法が表示されること。

- **再発防止（拙速 < 安全確実）**: マニュアル更新時は、**毎回**必読の「編集後の必須手順」をチェックリストとして実施する。AI や担当者に依頼する場合も「反映手順まで実施すること」を明示する（既に必読に記載済み）。

**追加のコード修正は不要**。上記手順で表示は更新される。

---

### 3.2 ヘルプチャットについて（根本解決・シンプル・具体的）

- **根本解決**: ヘルプチャットの回答に「CSV一括登録」と「有料代行の申し込み」が含まれるように、**FAQ データを追加・更新し、DB に反映し、キャッシュを無効化する**。  
- **シンプル**: 既存の `insert_operator_faqs.py` と `update_operator_faqs.py` の仕組みのまま、**1 件追加・1 件更新**のみ行う。  
- **具体的**: 以下を実施する。

#### 修正案 A: faq_bulk_import_paid_option の追加

- **ファイル**: `backend/scripts/insert_operator_faqs.py`（および定義書 `docs/help_system_faq_data.md`）。  
- **内容**: **faq_bulk_import** の直後（または faq_management カテゴリ内の適切な display_order）に、次の 1 件を **OPERATOR_FAQ_DATA** に追加する。
  - **intent_key**: `faq_bulk_import_paid_option`
  - **category**: `faq_management`
  - **display_order**: 81（faq_bulk_import が 80 のため）
  - **translations**:
    - **ja**: question「CSV一括登録の有料代行はありますか？どう申し込めますか？」、answer「はい、Standard・Premiumプラン向けに「CSV一括登録代行（有料オプション）」があります。FAQ管理画面の「CSV一括登録」ボタン近くの「代行をご希望の方はこちら」、またはサイドバーの「CSV代行の申し込み」から専用フォームへ進み、施設名・プラン・希望件数・希望言語・連絡先・FAQ原稿ファイル（.xlsx/.csv/.txt/.md）を送信してください。運営が料金表に沿って見積もりと決済案内を行い、合意後に開発者がCSV一括登録でFAQを登録します。」、keywords「CSV一括登録代行,有料オプション,代行申し込み,FAQ代行」、**related_url**: `/admin/csv-bulk-request`
    - **en**: 同趣旨の英文（question / answer / keywords / related_url）
- **定義書**: `docs/help_system_faq_data.md` に上記 intent_key の説明を追記する。

#### 修正案 B: faq_bulk_import の related_url 設定

- **ファイル**: `backend/scripts/insert_operator_faqs.py`。  
- **内容**: **faq_bulk_import** の `related_url` を **`/admin/faqs`** に設定する（FAQ 管理画面で CSV 一括登録モーダルを開けるため）。現状 `None` のため、誘導が弱い。

#### 修正案 C: DB 反映とキャッシュ無効化

- **DB 反映**: 上記 A・B の変更後、**ローカル**では次のいずれかを 1 回実行する。  
  - 既存レコードの更新のみ: `docker compose exec backend python scripts/update_operator_faqs.py`（プロジェクトルートで backend が起動している前提。DATABASE_URL は compose の postgres に合わせる）。  
  - 新規追加を含む場合: `update_operator_faqs.py` が intent_key で存在しなければ新規作成する実装であれば、それでよい。そうでなければ、手順に従い `insert_operator_faqs.py` の実行が必要かどうかを確認する（`docs/施設管理者向けヘルプチャットFAQ_マニュアル水準化_調査と計画.md` の「デプロイ時のDB反映」を参照）。  
- **キャッシュ無効化**: DB 反映後、**backend を再起動**する（`docker compose restart backend`）。または Redis の `operator_faqs:*` を削除する。これにより、ヘルプ API が最新の FAQ を返す。

## 4. 実施順序（修正案の実行順）

**指示があるまで実施しない。** 実施する場合は以下の順序を推奨する。

| 順 | 対象 | 内容 |
|----|------|------|
| 1 | **マニュアル表示** | 必須手順の実施（サイトデータ消去・SW 解除・ハードリロード）。コード変更なし。 |
| 2 | **ヘルプFAQデータ** | `insert_operator_faqs.py` に faq_bulk_import_paid_option を追加、faq_bulk_import の related_url を `/admin/faqs` に設定。`help_system_faq_data.md` を更新。 |
| 3 | **DB・キャッシュ** | ローカルで `update_operator_faqs.py` を実行し、backend を再起動（または Redis キャッシュ削除）。 |
| 4 | **確認** | マニュアル 4.8 の表示、ヘルプチャットで「CSV一括登録」「有料代行」「どう申し込めますか」等で期待どおりの回答とリンクが出ることを確認。 |
