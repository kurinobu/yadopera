## 2. 根本的な質問への回答

### 2.1 それぞれは何を目的として表示しているのか？

#### 2.1.1 総質問数カード

**目的**:
- **宿泊施設管理者に伝えたいこと**: 「どれくらいの質問が来ているか」
- **感じてほしいこと**: 
  - 質問の量を把握して、対応体制を考える材料にする
  - 質問数の増減傾向を把握する

**表示内容**:
- 過去7日間にゲストが送信した質問の総数

**ユーザーアクション**:
- 質問数が多い → 対応体制を見直す
- 質問数が少ない → サービスが順調に運営されている

#### 2.1.2 カテゴリ別内訳グラフ

**目的**:
- **宿泊施設管理者に伝えたいこと**: 「どのカテゴリのFAQがよく使われているか」
- **感じてほしいこと**:
  - よく使われるカテゴリのFAQを充実させる
  - 使われていないカテゴリのFAQを見直す

**表示内容**:
- 過去7日間にAIがFAQを使って回答した回数を、カテゴリ別に集計

**ユーザーアクション**:
- Basicが多い → 基本情報のFAQを充実させる
- Troubleが多い → トラブル対応のFAQを充実させる

### 2.2 なぜ同じセクションに配置されているのか？

**現在の設計思想（推測）**:
- 「週次サマリー」として、過去7日間のデータをまとめて表示
- しかし、実際には異なる指標を表示しているため、混乱を招いている

**問題点**:
- 総質問数は「質問の量」を示す
- カテゴリ別内訳は「FAQの使用頻度」を示す

## 4. 改善提案

### 4.1 ユーザーの提案

**提案内容**:
1. **2つを別セクションに配置して独立させる**
2. **グラフを廃止してスコア表示にする**
3. **カラーリングを削除する**

### 4.2 改善案

#### 4.2.1 セクション分離

**セクション1: 週次サマリー**（質問の量とAIの対応状況）
- 総質問数（過去7日間）
- 自動応答率（過去7日間）
- 平均信頼度（過去7日間）
- 未解決質問（全期間）

**セクション2: FAQ使用状況**（FAQの使用頻度）
- カテゴリ別使用回数（過去7日間）
  - Basic: X件
  - Facilities: Y件
  - Location: Z件
  - Trouble: W件
  - 合計: X+Y+Z+W件

#### 4.2.2 グラフの廃止とスコア表示への変更

**現在**: 円グラフで表示
**改善案**: スコア表示（カード形式）に変更

**理由**:
- グラフは視覚的に分かりやすいが、数値が正確に分からない
- スコア表示の方が、数値を正確に把握できる
- カラーリングを削除することで、他のセクションとの混同を防ぐ

#### 4.2.3 カラーリングの削除

**現在**: カテゴリごとに色分け（Basic: 青、Facilities: 緑、Location: 黄、Trouble: 赤）
**改善案**: カラーリングを削除し、テキストのみで表示

**理由**:
- カラーリングが他のセクション（総質問数カードなど）と混同される
- テキストのみの方が、シンプルで分かりやすい
