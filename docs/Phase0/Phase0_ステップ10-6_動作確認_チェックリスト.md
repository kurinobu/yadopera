## 確認項目

### 1. サイトアクセステスト

- [ ] `https://yadopera.com` でアクセス可能
- [ ] ページが正常に表示される（全セクションが表示される）
- [ ] スタイリングが正しく適用されている
- [ ] フォームが表示される
- [ ] ナビゲーションが動作する
- [ ] レスポンシブデザインが動作する（スマホ表示確認）

### 2. Google Analytics動作確認

#### 2-1. 開発者ツールで確認

1. ブラウザの開発者ツールを開く（F12）
2. Networkタブを開く
3. ページをリロード
4. 以下を確認：
   - [ ] `gtag/js?id=G-BE9HZ0XGH4` のリクエストが送信されている
   - [ ] ステータスが `200`（成功）である
   - [ ] CSPエラーがない

#### 2-2. Google Analyticsリアルタイムレポートで確認

1. [Google Analytics](https://analytics.google.com/) にアクセス
2. プロパティ「やどぺら LP」を選択
3. レポート → リアルタイム → 概要 を開く
4. 別のブラウザまたはタブで `https://yadopera.com` にアクセス
5. 以下を確認：
   - [ ] 「現在のユーザー数」が `1` 以上になっている
   - [ ] 「ページビュー」が記録されている
   - [ ] 「ページ」セクションに `/` が表示されている

### 3. フォーム動作確認

- [ ] PoC応募フォームが表示される
- [ ] フォームの各フィールドが入力可能
- [ ] フォーム送信が動作する（Formspree連携確認）

### 4. その他の確認

- [ ] コンソールエラーがない（警告は問題なし）
- [ ] ページの読み込み速度が適切
- [ ] すべてのリンクが動作する

## トラブルシューティング

### 問題: Google Analyticsが動作しない

**解決策**:
1. 測定IDが正しく設定されているか確認（`G-BE9HZ0XGH4`）
2. ブラウザの拡張機能（広告ブロッカー等）を無効化して試す
3. 別のブラウザで試す
4. Google Analyticsのリアルタイムレポートで確認（数分遅延する場合があります）

### 問題: ページが表示されない

**解決策**:
1. ブラウザのキャッシュをクリア（Cmd+Shift+R または Ctrl+Shift+F5）
2. 別のブラウザで試す
3. シークレットモードで試す
