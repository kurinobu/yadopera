## 次のステップ（推奨順序）

### 最優先（即時実施）

#### ステップ10-2: Vercelにランディングページをデプロイ

**優先度**: 高（最優先）  
**所要時間**: 30分  
**前提条件**: なし（すぐに実施可能）

**手順**:
1. Vercelアカウント作成（未作成の場合）
   - https://vercel.com/ にアクセス
   - GitHubアカウントでログイン
2. GitHubリポジトリ連携
   - Vercelダッシュボードで「New Project」をクリック
   - GitHubリポジトリ（`kurinobu/yadopera`）を選択
3. プロジェクト設定
   - **Root Directory**: `landing` を指定
   - **Framework Preset**: Other（またはStatic Site）
   - **Build Command**: なし（静的HTMLのため）
   - **Output Directory**: `.`（ルートディレクトリ）
4. デプロイ実行
   - 「Deploy」をクリック
   - デプロイ完了を待つ（1-2分）

**完了基準**:
- [ ] Vercelデプロイ完了
- [ ] VercelのデフォルトURL（例: `yadopera.vercel.app`）でアクセス可能
- [ ] ページが正常に表示される

**注意**: この時点ではまだカスタムドメイン（yadopera.com）は設定していない。

#### ステップ10-3: カスタムドメイン設定

**優先度**: 高  
**所要時間**: 10分（DNS設定含む）  
**前提条件**: ステップ10-2完了

**手順**:
1. Vercelプロジェクト設定を開く
2. 「Domains」タブを選択
3. `yadopera.com` を追加

   - ドメイン管理画面でDNSレコードを設定
5. SSL証明書は自動で設定される（数分かかる場合あり）

**完了基準**:
- [ ] カスタムドメイン設定完了
- [ ] `https://yadopera.com` でアクセス可能
- [ ] SSL証明書が有効

**重要**: この時点で `https://yadopera.com` が利用可能になる。これがGoogle Analytics 4プロパティ作成時に必要なURL。

#### ステップ10-4: Google Analytics 4プロパティ作成

**優先度**: 高  
**所要時間**: 30分  
**前提条件**: 
- ✅ ステップ10-2完了（Vercelデプロイ完了）
- ✅ ステップ10-3完了（カスタムドメイン設定完了）
- ✅ `https://yadopera.com` が利用可能

**手順**:
1. [Google Analytics](https://analytics.google.com/)にアクセス
2. 「管理」→「プロパティを作成」
3. プロパティ設定:
   - プロパティ名: 「やどぺら LP」
   - **ウェブサイトURL**: `https://yadopera.com`（実際のURL）
   - レポートのタイムゾーン: 「日本標準時」
   - 通貨: 「日本円 (¥)」
4. 「作成」をクリック
5. 「管理」→「データストリーム」→「ウェブ」を選択
6. 測定ID（`G-XXXXXXXXXX`形式）をコピー

**完了基準**:
- [ ] Google Analytics 4プロパティ作成完了
- [ ] 測定ID取得済み（`G-XXXXXXXXXX`形式）

**注意**: Render.comのサービス作成は不要（ランディングページはVercelにデプロイするため）

---

#### ステップ10-5: HTMLに測定ID設定

**優先度**: 高  
**所要時間**: 10分  
**前提条件**: ステップ10-4完了（測定ID取得済み）

**手順**:
1. `landing/index.html`を編集
2. 26行目と31行目の`G-XXXXXXXXXX`を実際の測定IDに置き換え
   ```html
   <!-- 変更前 -->
   <script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
   gtag('config', 'G-XXXXXXXXXX', {
   
   <!-- 変更後 -->
   <script async src="https://www.googletagmanager.com/gtag/js?id=G-実際の測定ID"></script>
   gtag('config', 'G-実際の測定ID', {
   ```
3. コミット・プッシュ
   ```bash
   git add landing/index.html
   git commit -m "Add Google Analytics measurement ID"
   git push origin main
   ```
4. Vercelが自動で再デプロイ（1-2分）

**完了基準**:
- [ ] HTMLに測定ID設定完了
- [ ] コミット・プッシュ完了
- [ ] Vercel再デプロイ完了
