## 1. 問題の概要

### 1.1 発生した問題

**問題**: データ不整合の問題を「残存課題」として誤って報告した

**症状**:
- 引き継ぎ書には「データ不整合の問題は解決済み（ステップ1完了）」と明記されていた
- 引き継ぎ書の「10.2 残存課題」には「合計: 3項目」と記載されていた
- しかし、参照文書（`docs/Phase1_Phase2_残存課題_完全調査分析_ステップ計画_20251217.md`）の内容を優先して、データ不整合の問題を「残存課題」として報告した

**影響**:
- 時間とトークンの浪費
- ユーザーのストレス
- 次のステップに即座に進めない

### 1.2 根本原因

1. **引き継ぎ書の全文精読不足**: 引き継ぎ書を全文精読せずに、参照文書の内容を優先した
2. **時刻を含む最新情報の確認不足**: 引き継ぎ書の最終更新日時（時刻を含む）を確認せずに、古い参照文書の内容を参照した
3. **整合性チェックの欠如**: 引き継ぎ書と参照文書の矛盾を確認せずに報告した
4. **セッション開始チェックリストの未実行**: セッション開始時にチェックリストを実行しなかった

## 2. 厳重再発防止策

### 2.1 セッション開始チェックリストの必須実行

**ルール**: **セッション開始時、最初に必ずセッション開始チェックリストを実行する**

**手順**:
1. セッション開始時、**最初に**`docs/Phase1_Phase2_セッション開始チェックリスト_厳重版_20251217.md`を開く
2. セクション1から順番に実行
3. すべてのチェック項目にチェックを入れる
4. 報告形式に従って報告する
5. **報告完了後に作業を開始する**

**禁止事項**:
- チェックリストを完了する前に作業を開始すること
- チェックリストを省略すること
- チェックリストの一部のみを実行すること

**関連文書**: `docs/Phase1_Phase2_セッション開始チェックリスト_厳重版_20251217.md`

---

### 2.2 最新引き継ぎ書の特定と全文精読（必須）

**ルール**: **引き継ぎ書を最初から最後まで全文精読する**

**手順**:
1. `docs/`ディレクトリ内で`Phase1_Phase2_引き継ぎ書_*.md`を検索
2. **時刻を含む最終更新日時が最新のファイルを特定**
3. そのファイルを**最初から最後まで全文精読**
4. 重要なセクションを確認:
   - 「作成日時」または「最終更新日時」（時刻を含む）
   - 「状態」欄
   - 「本セッション追記」セクション
   - 「1. 現在の進捗状況」セクション
   - 「10.2 残存課題」セクション
   - 「10.3 次のセッションで実施すべき作業」セクション
   - 「8. 環境情報」セクション

**重要**: 
- 「精読」とは、**1文字1文字読み、内容を理解すること**。流し読みは禁止。
- 日付だけでなく、**時刻も確認**すること。同じ日付でも時刻が異なる場合がある。

---

### 2.3 参照文書の時刻確認と整合性チェック（必須）

**ルール**: **引き継ぎ書の最終更新日時より古い参照文書の内容は、引き継ぎ書の最新情報を優先する**

**手順**:
1. 引き継ぎ書内で参照されている文書を特定
2. 各参照文書の作成日時・最終更新日時（時刻を含む）を確認
3. 引き継ぎ書の最終更新日時と比較
4. 引き継ぎ書の最終更新日時より古い参照文書の内容は、**引き継ぎ書の最新情報を優先**する
5. 矛盾がある場合は、引き継ぎ書を優先する

**チェック項目**:
- [ ] 引き継ぎ書内で参照されている文書を特定したか
- [ ] 各参照文書の作成日時・最終更新日時（時刻を含む）を確認したか
- [ ] 引き継ぎ書の最終更新日時より古い参照文書を特定したか
- [ ] 矛盾がある場合は、引き継ぎ書を優先すると判断したか

---

### 2.4 残存課題の正確な把握（必須）

**ルール**: **引き継ぎ書の「10.2 残存課題」セクションの内容を最優先で確認する**

**手順**:
1. 引き継ぎ書の「10.2 残存課題」セクションを精読
2. 残存課題の数を確認（引き継ぎ書に記載されている数）
3. 各残存課題の状態を確認
4. 解決済みの問題が残存課題に含まれていないか確認
5. 参照文書の残存課題リストと比較し、矛盾がある場合は引き継ぎ書を優先

**チェック項目**:
- [ ] 引き継ぎ書の「10.2 残存課題」セクションを精読したか
- [ ] 残存課題の数を確認したか（引き継ぎ書に記載されている数）
- [ ] 各残存課題の状態を確認したか
- [ ] 解決済みの問題が残存課題に含まれていないか確認したか
- [ ] 参照文書の残存課題リストと比較したか
- [ ] 矛盾がある場合は、引き継ぎ書を優先すると判断したか

### 2.5 文書作成日時の絶対時刻入力（必須）

**ルール**: **すべての文書の作成日時・更新日時には絶対時刻（時分秒）を入力する**

**形式**:
- **日本語形式**: `YYYY年MM月DD日 HH時MM分SS秒`

**時刻取得コマンド**:

**チェック項目**:

- [ ] 更新日時に絶対時刻（時分秒）を入力したか（更新の場合）
- [ ] 時刻取得コマンドを実行して正確な時刻を取得したか

## 3. 禁止事項

### 3.1 絶対に禁止される行為

1. **チェックリストを完了する前に作業を開始すること**
2. **引き継ぎ書を全文精読せずに報告すること**
3. **参照文書の内容を引き継ぎ書より優先すること**
4. **解決済みの問題を残存課題として報告すること**
5. **完了済みの作業を再度実施しようとすること**
6. **時刻を確認せずに最新文書を特定すること**
7. **矛盾を確認せずに報告すること**
8. **引き継ぎ書の「10.2 残存課題」セクションを確認せずに残存課題を報告すること**

## 4. 違反時の対応

### 4.1 チェックリスト未完了で作業を開始した場合

**対応**:
1. 即座に作業を停止する
2. チェックリストを完了する
3. 完了後に作業を再開する

### 4.2 誤った報告をした場合

**対応**:
1. 即座に謝罪する
2. 原因を調査分析する
3. 再発防止策を強化する
4. 誤報告の原因とエビデンスを明らかにして報告する
