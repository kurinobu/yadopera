## 3. A実装内容（縦位置を上下中央に）

**対象ファイル**: `backend/app/services/qr_code_service.py`

- **PNG**
  - テキスト描画を画像中央 `(center_x, center_y)` にし、Pillow の `anchor="mm"` で上下左右中央に配置。
  - 白背景矩形も中央基準で描画。TrueType でない場合は従来の `text_x, text_y` で描画。
- **PDF**（QR画像部分）
  - 上記と同様に `anchor="mm"` で中央配置、フォールバック付き。
- **SVG**
  - `<text>` に `dominant-baseline="middle"` を指定し、`y="{center_y}mm"` に変更（従来の `y="{center_y + font_size/3}mm"` を削除）。
- **例外処理**
  - フォント読み込みの `except` を `(OSError, IOError, FileNotFoundError)` に限定。

---

## 4. B実装内容（Notoフォント）

**対象**: `backend/Dockerfile`, `backend/app/services/qr_code_service.py`

- **Dockerfile**
  - `fonts-noto-core` を `apt-get install` に追加（`fonts-noto-cjk` の前）。
- **qr_code_service.py**
  - 定数 `YADOPERA_FONT_PATHS` を追加: 先頭に `/usr/share/fonts/truetype/noto/NotoSans-Bold.ttf`、次に `DejaVuSans-Bold.ttf`。
  - PNG・PDF の「YadOPERA」描画で上記リストを順に試し、最初に読み込めたフォントを使用。どれも失敗した場合は `ImageFont.load_default()`。
  - SVG の `<text>` の `font-family` を `"Noto Sans, sans-serif"` に変更。

**注意**: PDF の日本語（施設名・設置場所・生成日時）は従来どおり IPA ゴシック（.ttf）のみ使用。Noto CJK（.ttc）は reportlab に渡していない。

## 6. 今後の実施者向け

- **ローカルでA確認**: `docker-compose up -d` のまま、管理画面で QR プレビュー（PNG/PDF/SVG）を開き、「YadOPERA」が中央余白の上下中央にあることを確認。
- **ステージングでA確認**: デプロイ後、同様にプレビューで縦位置を確認。
- **ローカルでB確認**: `docker-compose build backend` でバックエンドを再ビルドしてから `docker-compose up -d`。QR の「YadOPERA」が Noto Sans 風であることと、縦位置が中央のままであることを確認。
- **ステージングでB確認**: B をデプロイしたうえで、同様の項目をステージングで確認。

## 7. SVG縦位置の追加修正（2026-03-04・ローカルテスト後）

**事象**: ローカルブラウザテストで、SVGのみ「YadOPERA」が上下中央より**上に行き過ぎ**て表示される。

**対応**: `dominant-baseline="middle"` のまま、テキストの `y` に下方向オフセット `font_size * 0.2` を加算。  
`text_y_svg = center_y + font_size * 0.2` とし、`y="{text_y_svg}mm"` で描画。

**バックアップ**: `backups/20260304_qr_svg_vertical_fix/qr_code_service.py`（修正前）
