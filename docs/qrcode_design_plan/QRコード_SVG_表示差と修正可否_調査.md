# QRコード SVG「YadOPERA」— 表示差と修正可否の調査

**作成日**: 2026年3月4日  
**目的**: 開き方（Finderサムネイル vs ブラウザ）によって縦位置・横幅が変わる事象を整理し、修正の可否と選択肢をまとめる。**指示があるまで実装は行わない。**

---

## 1. 事象の整理（スクショ・報告より）

| 表示環境 | 縦位置 | 横幅 |
|----------|--------|------|
| **ブラウザ（Safari等）** | 期待どおり（上下中央） | 「Y」「A」が白い余白の左右にはみ出る |
| **Finder サムネイル／プレビュー** | 中心より若干上 | 余白ギリギリで収まっている |

---

## 2. 表示が変わる主な理由

1. **フォントの解釈が環境ごとに違う**
   - SVG では `font-family="Noto Sans, sans-serif"` を指定しているが、**フォントは SVG に埋め込んでいない**。
   - ブラウザ: システムの Noto Sans や別の sans-serif を使用し、メトリクス（文字幅・ベースライン）がその環境依存。
   - Finder プレビュー: macOS の別レンダラを使うため、別のフォント・スケールになることがある。
   - その結果、**同じ `font-size` でも「見た目の文字幅・高さ」が変わる**。

2. **縦位置の差**
   - `dominant-baseline="middle"` と `y` の解釈がレンダラごとに微妙に異なる場合がある。
   - ブラウザ用に `y` にオフセット（`font_size * 0.2`）を入れているため、Finder では「やや上」に見える可能性がある。

3. **横幅のはみ出し（ブラウザ）**
   - 白矩形の幅は **`rect_width = width / 3`**（例: 57mm なら 19mm）で固定。
   - テキスト幅は **指定していない**（`font-size` のみ）。ブラウザで Noto Sans Bold などで描画すると、「YadOPERA」の実幅が 19mm を超えることがある。
   - そのため、中央揃えのまま左右（特に「Y」と「A」）がはみ出す。

---

## 3. 修正は可能か・大きさの固定化は可能か

### 3.1 結論（短く）

- **表示差を「なくす」ことは、SVG の仕様とフォント未埋め込みの前提では不可能**です。  
  レンダラごとの解釈差は完全には揃えられません。
- **はみ出しの解消**や**ある程度の見た目統一**は、以下のような**トレードオフ付きの対応**で可能です。

### 3.2 横幅のはみ出しを減らす／なくす選択肢

| 方針 | 内容 | メリット | デメリット／注意 |
|------|------|----------|------------------|
| **A. 白矩形を広くする** | `rect_width` を大きくする（例: `width / 2.5` や `width / 2.2`）。 | ブラウザではみ出しにくくなる。 | 白領域が大きくなり、QR の誤り訂正に依存するため、極端にすると読み取りに影響する可能性がある。 |
| **B. フォントサイズを小さくする** | `font_size = width / 18` などに変更。 | 多くの環境で白矩形内に収まりやすくなる。 | サムネイルなどでは文字がより小さく見える。 |
| **C. textLength で幅を強制** | `<text textLength="…" lengthAdjust="spacing">` で、テキストの**合計幅**を白矩形幅に合わせる。 | レンダラが対応していれば、**同じ SVG で「文字列の幅」は一定**にできる。 | 字間だけが伸縮し、デザインがやや不自然になる可能性。ブラウザは対応が多いが、Finder プレビュー等の対応は要確認。 |

### 3.3 縦位置の差について

- 現状、**ブラウザを主な利用想定**としてオフセットを入れているため、**ブラウザで期待どおり**になっている。
- Finder で「やや上」になるのは、**レンダラ差**によるもの。  
  一つの `y` 値で「ブラウザも Finder も完全に同じ」にすることは、フォントを埋め込まない限り現実的には困難。
- 対応方針の例:
  - **ブラウザを正とする**: 現状維持で、文書で「プレビュー環境により縦位置が多少異なる場合がある」と注記する。
  - **オフセットを弱める**: オフセットを小さくして、Finder に少し寄せる代わりに、ブラウザでわずかに上にずれる可能性を許容する。

### 3.4 「大きさを固定化」できる範囲

- **SVG 内の寸法（mm）は固定**している（`width="57mm"`、`rect` の幅・高さ、`font-size` など）。
- **「見た目の文字の幅・高さ」は、使用フォントとレンダラに依存する**ため、**すべての開き方で同一にすることは不可能**です。
- **可能なこと**:
  - **textLength** を使うと、**「テキスト全体の幅」だけ**を指定幅に揃えられる（字間で調整）。対応しているビューアでは、開き方による「幅の差」は小さくなる。
  - 白矩形を広げる／フォントサイズを下げることで、「多くの環境ではみ出さない」ようにすることはできる。

---

## 4. 実装する場合の推奨案（指示があるまで実施しない）

- **はみ出し対策を優先する場合**
  - **A（白矩形を広くする）** と **B（フォントサイズを少し下げる）** の併用が安全。
  - 例: `rect_width = width / 2.5`、`font_size = width / 17` など。必要なら値は微調整。
- **「幅」の再現性をより揃えたい場合**
  - **C（textLength）** を追加。  
    `textLength="{rect_width}mm"` および `lengthAdjust="spacing"` を `<text>` に指定。  
    対応環境では、テキストの総幅が白矩形幅に一致し、はみ出しがなくなると期待できる。
- **縦位置**
  - 主利用をブラウザとするなら現状維持。  
    Finder との一致を優先するなら、オフセットを `font_size * 0.1` などに小さくする検討。

## 5. まとめ

| 質問 | 回答 |
|------|------|
| 開き方による表示差は修正できるか？ | **完全に揃えることはできない**。レンダラ・フォント差のため。 |
| 横幅のはみ出しは修正できるか？ | **できる**。白矩形の拡大・フォントサイズの縮小・`textLength` のいずれか（または組み合わせ）で、はみ出しを防げる。 |
| 大きさを固定化することは可能か？ | **寸法（mm）は既に固定**。**見た目の文字の大きさ・幅**は環境依存のため、**完全な固定は不可能**。`textLength` で「テキスト全体の幅」だけは一定にできる。 |
