## 7. C1実施手順（Runbook §1.2 準拠、MVP短縮版）

1. **Render Backend（本番）を作成**
   - Service: Web Service（Docker）
   - Name: `yadopera-backend-production`
   - Branch: `main`
   - Dockerfile: `backend/Dockerfile`
   - Pre-deploy: `alembic upgrade head`
2. **Render Frontend（本番）を作成**
   - Service: Static Site
   - Name: `yadopera-frontend-production`
   - Branch: `main`
   - Root/Build: `frontend` / `npx vite build`
3. **Backend 環境変数を投入（§4.1）**
   - `ENVIRONMENT=production`
   - `CORS_ORIGINS=https://yadopera-frontend-production.onrender.com`
   - `FRONTEND_URL=https://yadopera-frontend-production.onrender.com`
   - `STRIPE_*` は **test** 値を投入（live は順7）
4. **Frontend 環境変数を投入（§4.2）**
   - `VITE_API_BASE_URL=https://yadopera-backend-production.onrender.com`
   - `VITE_ENVIRONMENT=production`
5. **初回デプロイ検証**
   - Backend `/api/v1/health` が 200
   - `alembic upgrade head` 実行ログ確認
   - `GET /__debug_env` が 404/非公開
6. **接続先実測**
   - DB: `current_database()/host/port` を記録（本番DB接続）
   - Stripe: test `sk_test` と `whsec` が同一 `acct_` を確認
7. **記録更新**
   - 本書 §2「状態」を更新
   - `docs/20260415_サービス開始ロードマップ.md` の C1 を完了化
   - 実測証跡を `docs/evidence/` に追記
