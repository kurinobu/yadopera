# 2026-03-18 手動返信機能 Phase 5 ローカル検証手順

## 1. 起動（Docker）

以下が起動していること（`docker compose ps`）:
- backend: `0.0.0.0:8000->8000`
- frontend: `0.0.0.0:5173->5173`
- postgres / redis: healthy

起動コマンド（未起動の場合）:
- `docker compose up -d --build postgres redis backend frontend`

## 2. ブラウザで開くURL

- **フロントトップ**: `http://localhost:5173/`
- **管理者ログイン**: `http://localhost:5173/admin/login`
- **新規登録**: `http://localhost:5173/admin/register`
- **管理ダッシュボード**: `http://localhost:5173/admin/dashboard`
- **会話詳細**: `http://localhost:5173/admin/conversations/<session_id>`

ゲスト:
- **言語選択**: `http://localhost:5173/f/<facilityId>`
- **Welcome**: `http://localhost:5173/f/<facilityId>/welcome`
- **チャット**: `http://localhost:5173/f/<facilityId>/chat`

## 3. 検証シナリオ（E2E）

### 3.1 ゲスト側でエスカレーションを発生させる

1. ゲスト画面を開く: `http://localhost:5173/f/<facilityId>`
2. 言語選択 → Welcome → Chat に遷移
3. チャットで質問を送信
4. 「スタッフに連絡 / Contact Staff」ボタンでエスカレーションを作成

期待:
- エスカレーションが作成され、管理画面ダッシュボードの「未解決のエスカレーション」に表示される

### 3.2 管理画面で手動返信する

1. 管理画面にログイン（`/admin/login`）
2. ダッシュボードの「未解決のエスカレーション」から該当会話を開く
3. 会話詳細の「ゲストに返信」に返信文を入れて送信

期待:
- 会話詳細のメッセージ一覧に `role=staff` のメッセージが追加される（ラベル「スタッフ」）

### 3.3 ゲスト側でスタッフ返信が見えること

1. ゲスト側のチャット画面をリロード（または同一 session で再表示）

期待:
- スタッフ返信が左寄せで表示され、吹き出し内に「スタッフ」ラベルが出る
- 信頼度表示・フィードバックUIがスタッフ返信には出ない

### 3.4 解決を促すUIと解決処理

1. 会話詳細に「未解決のエスカレーション」ブロックが表示されていることを確認
2. 促し文言が表示されていることを確認
3. 「解決する」ボタンを押す

期待:
- 解決済みフィードバックが表示される
- 該当エスカレーションが会話詳細の未解決リストから消える
- ダッシュボードの未解決一覧からも消える（再取得後）

## 4. 備考（反映されない場合）

- PWA/Service Worker の影響が疑わしい場合は、`docs/マニュアル更新・改訂_必読.md` の手順に従い、サイトデータ消去・SW解除・ハードリロードを実施する。
