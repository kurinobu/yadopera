# パスワードリセット機能 — ブラウザテスト手順

パスワードリセット実装（Step 1〜10 完了）の動作確認用チェックリストです。ローカル環境で Docker を起動した状態でブラウザテストを実施してください。

---

## 1. 事前準備

### 1.1 Docker でフルスタック起動（推奨）

```bash
cd /Users/kurinobu/projects/yadopera
docker-compose up -d
```

- 未ビルド時: `docker-compose build` を先に実行
- 起動確認: バックエンド http://localhost:8000 、フロント http://localhost:5173 が応答すること

### 1.2 フロントのみで確認する場合（API は未使用）

```bash
cd frontend && npm run dev
```

- ログイン・リセット送信・確定はバックエンドが必要なため、Docker でバックエンドも起動した状態でテストすることを推奨

### 1.3 アクセス URL

| 用途           | URL                          |
|----------------|------------------------------|
| 管理画面ログイン | http://localhost:5173/admin/login |
| パスワードリセット依頼 | http://localhost:5173/admin/password-reset |
| パスワードリセット確定 | http://localhost:5173/admin/password-reset/confirm?token=**トークン** |
| API ドキュメント | http://localhost:8000/docs   |

## 2. 確認項目チェックリスト

実施したらチェックを入れてください。

### (1) ログイン画面の導線

- [ ] http://localhost:5173/admin/login を開く
- [ ] 「パスワードを忘れた場合はこちら」リンクが表示されている
- [ ] リンクをクリックすると `/admin/password-reset` に遷移する

### (2) パスワードリセット依頼画面

- [ ] パスワードリセット依頼画面で「YadOPERA」「パスワードリセット / Password Reset」が表示される
- [ ] メールアドレス入力欄と「リセット用メールを送信」ボタンがある
- [ ] 「ログインに戻る」リンクで `/admin/login` に戻れる

### (3) リセット用メール送信

- [ ] 登録済みメールアドレスを入力して「リセット用メールを送信」をクリック
- [ ] 成功時「送信しました。メールをご確認ください。」が表示される
- [ ] 未登録メールでも同じ成功メッセージになる（情報漏れ防止）

### (4) レート制限（60秒）

- [ ] 同じメールアドレスで 60 秒以内に再度「リセット用メールを送信」を実行
- [ ] フロントに「しばらく待ってから再度お試しください。」（または 429 に相当するメッセージ）が表示される

### (5) リセット URL の取得（Brevo 未設定時）

- [ ] バックエンドログでリセット用 URL を確認する  
  `docker-compose logs -f backend` でトークン付き URL を探す
- [ ] または Brevo 設定済みの場合は、送信されたメール内のリンクをクリック

### (6) パスワードリセット確定画面

- [ ] `/admin/password-reset/confirm?token=取得したトークン` にアクセス
- [ ] 「新しいパスワード」「新しいパスワード（確認）」入力欄と「パスワードを変更」ボタンがある
- [ ] 8 文字以上の同じパスワードを入力して「パスワードを変更」をクリック
- [ ] 「パスワードを変更しました。ログインしてください。」が表示される
- [ ] 「ログインページへ」で `/admin/login` に遷移する

### (7) 新パスワードでログイン

- [ ] ログイン画面で、リセットしたメールアドレスと新しいパスワードでログインできる

### (8) トークン再利用で 400

- [ ] 同じリセット用 URL（同じ token）で再度「パスワードを変更」を実行
- [ ] 無効トークン扱いとなり「リンクが無効または期限切れです。再度リセット用メールを請求してください。」が表示される（または API 400 のメッセージ）

### その他

- [ ] token なしで `/admin/password-reset/confirm` にアクセスすると「リンクが無効または期限切れです…」が表示される
- [ ] パスワード不一致・8 文字未満でフォームエラーが表示される

## 3. メール未設定（Brevo 未設定）の場合

- リセット依頼は 200 で成功するが、メールは送信されない
- トークン・URL はバックエンドログに出力される場合があるため、ログから URL をコピーして (5)(6)(7) を手動で検証する

---

## 4. トラブルシュート

| 現象 | 確認すること |
|------|----------------|
| ログイン画面にリンクが出ない | キャッシュ削除、フロント再ビルド `npm run build` |
| 送信後も「送信しました」にならない | ブラウザ DevTools の Network で API レスポンスを確認、バックエンドログ `docker-compose logs backend` |
| 429 が出ない | 60 秒以内に同じメールで再送しているか、バックエンドのレート制限が有効か確認 |
| 確定画面で 400 | token の有効期限（1 時間）切れ、または既に使用済み |

---

## 5. テスト完了後

- Docker を止める: `docker-compose down`
- ログを見る: `docker-compose logs backend` / `docker-compose logs frontend`
