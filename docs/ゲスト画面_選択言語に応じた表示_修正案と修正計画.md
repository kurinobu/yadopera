# ゲスト画面 — 選択言語に応じた表示の修正案と修正計画

**作成日**: 2026年3月11日  
**課題**: どの言語を選択してもウェルカム／チャット周りの画面が変わらず、URL の `lang` だけが変わる状態の解消。緊急連絡先のラベルが日本語固定で緊急時に対応できていない問題を含む。  
**スコープ**: プランに依存しないゲスト画面の表示文言の多言語化。  
**方針**: **指示があるまで実装しない。**

---

## 1. 問題の整理

### 1.1 事象（3スクショで確認された内容）

- `?lang=fr` / `?lang=zh-CN` / `?lang=ko` でウェルカム画面を開いても、表示される文言がほぼ同じ。
- 変わっているのは URL のみで、以下は**選択言語に依存せず固定**（主に日本語または日英併記）:
  - セクション見出し「よくある質問 / Frequently Asked Questions」
  - FAQ 3件のテキスト（※施設が該当言語の FAQ を登録していれば API はその言語で返すが、登録がなければ en フォールバックで英語表示になる）
  - メッセージ入力「メッセージを入力 / Enter your message」「質問を入力してください...」
  - 緊急連絡先「緊急連絡先 / Emergency Contact」「救急・消防」「警察」「施設連絡先」
  - 施設ヘッダーの「Check-in」「Check-out」（英語固定）

### 1.2 問題の本質

- ゲスト画面の**UIラベル・見出し・プレースホルダ**が、ゲストの選択言語（`route.query.lang`）で切り替わっていない。
- 緊急連絡先は**緊急時に誰でも意味が分かる**必要があるため、選択言語での表示が特に重要。

---

## 2. 大原則との対応

| 大原則 | 本件での適用 |
|--------|----------------|
| **根本解決 > 暫定解決** | 文言を「ゲストが選択した言語」で一貫して表示する。緊急連絡先は緊急時でも理解できるようにする。 |
| **統一・同一化 > 特殊独自** | 既存の `couponCopy.ts` と同様に、「言語コード → 表示文言」のユーティリティを1つにまとめ、ゲスト用文言を同じパターンで管理する。 |
| **拙速 < 安全確実** | 対応言語は `SUPPORTED_LANGUAGES` と揃え（ja, en, zh-TW, zh-CN, fr, es, ko, th）、未対応は en にフォールバックする。 |

## 3. 対象箇所一覧（ゲスト画面で言語切り替えすべき文言）

### 3.1 ウェルカム画面（Welcome.vue とその子コンポーネント）

| コンポーネント | ファイル | 現状の文言（固定） | 備考 |
|----------------|----------|---------------------|------|
| **TopQuestions** | `TopQuestions.vue` | 「よくある質問 / Frequently Asked Questions」 | セクション見出し |
| 同上 | 同上 | 「よくある質問はありません」 | FAQ 0件時 |
| **MessageInput** | `MessageInput.vue` | 「メッセージを入力 / Enter your message」 | ラベル |
| 同上 | 同上 | 「質問を入力してください...」 | placeholder |
| **EmergencyContact** | `EmergencyContact.vue` | 「緊急連絡先 / Emergency Contact」 | 見出し |
| 同上 | 同上 | 「救急・消防」「警察」「施設連絡先」 | ラベル（緊急時対応の要） |
| **FacilityHeader** | `FacilityHeader.vue` | 「Check-in:」「Check-out:」 | 施設ヘッダー |
| 同上 | 同上 | 「Loading...」 | 施設未取得時 |

### 3.2 チャット画面（Chat.vue）

- 「トークン統合 / Link」「読み込み中...」「AIが回答を生成中...」「メッセージを入力...」「エラーが発生しました」等も日本語または日英固定。
- **本計画の第一次スコープはウェルカム画面**とする。チャット画面の文言は、同じ仕組み（後述の `guestCopy`）を拡張する形で**別ステップ**で対応可能とする。

---

## 4. 修正方針

### 4.1 一元化する「ゲスト用コピー」の構成

- **新規ファイル**: `frontend/src/utils/guestCopy.ts`
  - 役割: ゲスト画面で使う**全ての表示文言**を、言語コード別に返す。
  - 既存の `couponCopy.ts` と同様に、`getGuestCopy(lang: string)` のような関数で 1 オブジェクトを返し、未対応言語は `en` にフォールバックする。
- **対象文言**（第一次）:
  - **緊急連絡先**: title, ambulanceFire, police, facilityContact
  - **よくある質問**: sectionTitle, emptyMessage
  - **メッセージ入力**: label, placeholder
  - **施設ヘッダー**: checkIn, checkOut, loading

### 4.2 対応言語

- `SUPPORTED_LANGUAGES` と同一: **ja, en, zh-TW, zh-CN, fr, es, ko, th** の 8 言語。
- 上記以外の `lang` の場合は **en** を返す。

### 4.3 コンポーネント側の変更方針

- 各コンポーネントは **props で `lang: string` を受け取る**（省略時は `'en'`）。
- `guestCopy.ts` の `getGuestCopy(lang)` の戻り値から必要なキーを参照して表示する。
- **Welcome.vue**: `route.query.lang` を取得し、TopQuestions / MessageInput / EmergencyContact / FacilityHeader に `lang` を渡す。
- FacilityHeader は現状 `Welcome.vue` とチャットの両方で使われうるため、`lang` を optional とし、無い場合は `'en'` 扱いとする。

## 5. 修正計画（実施ステップ）

### Step 1: ゲスト用コピーモジュールの新規作成

| 項目 | 内容 |
|------|------|
| ファイル | `frontend/src/utils/guestCopy.ts` |
| 内容 | 8 言語分のオブジェクトと `getGuestCopy(lang: string)` を定義。戻り値に emergency, topQuestions, messageInput, facilityHeader をまとめて含める。 |
| 文言 | 緊急連絡先・よくある質問・メッセージ入力・施設ヘッダー用のキーを上記のとおり定義。各言語の文言は既存の `emergencyContactCopy` 案（別ドキュメント）および一般的な表記に合わせる。 |

### Step 2: ウェルカム画面の子コンポーネント修正

| コンポーネント | 変更内容 |
|----------------|----------|
| **EmergencyContact.vue** | props に `lang?: string` を追加。`getGuestCopy(lang).emergency` で見出し・ラベルを表示。119/110/施設電話は従来どおり。 |
| **TopQuestions.vue** | props に `lang?: string` を追加。`getGuestCopy(lang).topQuestions.sectionTitle` / `emptyMessage` で表示。 |
| **MessageInput.vue** | props に `lang?: string` を追加。`getGuestCopy(lang).messageInput.label` / `placeholder` で表示。 |
| **FacilityHeader.vue** | props に `lang?: string` を追加。`getGuestCopy(lang).facilityHeader.checkIn` / `checkOut` / `loading` で表示。 |

### Step 3: Welcome.vue の修正

- `language = route.query.lang || 'en'` をそのまま利用。
- `<FacilityHeader :facility="..." :lang="language" />`
- `<TopQuestions :questions="..." :lang="language" @question-click="..." />`
- `<MessageInput :lang="language" ... />`
- `<EmergencyContact :facility-phone="..." :lang="language" />`

### Step 4: 動作確認

- ウェルカム画面を `?lang=ja` / `en` / `fr` / `zh-CN` / `ko` 等で開き、見出し・ラベル・プレースホルダ・緊急連絡先が選択言語に応じて切り替わることを確認する。
- 未対応の `lang` で en にフォールバックすることを確認する。

### Step 5（任意）: チャット画面の多言語化

- Chat.vue 内の「読み込み中...」「AIが回答を生成中...」「メッセージを入力...」「エラーが発生しました」「トークン統合 / Link」等を、`guestCopy.ts` にキーを追加し、`route.query.lang` を子に渡して表示する。  
- 第一次では必須とせず、ウェルカムの動作確認後に別ステップで実施可能とする。

## 6. 既存ドキュメントとの関係

- **緊急連絡先のみ**の修正案は `docs/ゲスト画面_言語カード並び順と緊急連絡先多言語_調査報告と修正計画.md` に記載済み。
- 本ドキュメントは「**選択言語に応じて画面全体の表示が切り替わる**」ようにするため、緊急連絡先を含め**ウェルカムまわりの文言を一括**で `guestCopy.ts` に集約する修正案・修正計画とする。  
- 実装時は、緊急連絡先を単独の `emergencyContactCopy.ts` にするか、本計画どおり `guestCopy.ts` に統合するかをどちらかに揃える（**統一・同一化**のため 1 モジュールにまとめることを推奨）。

---

## 7. まとめ

- **修正案**: ゲストの選択言語（`route.query.lang`）に応じて、ウェルカム画面の見出し・ラベル・プレースホルダ・緊急連絡先・施設ヘッダー文言を切り替える。  
- **実装方針**: `frontend/src/utils/guestCopy.ts` を新規作成し、8 言語分のコピーを返す。各コンポーネントは `lang` を受け取り `getGuestCopy(lang)` を参照する。  
- **修正計画**: Step 1 で guestCopy 作成 → Step 2 で各コンポーネントに `lang` 対応 → Step 3 で Welcome から `lang` を渡す → Step 4 でブラウザ確認。緊急連絡先は本計画に含まれるため、既存の緊急連絡先のみの計画と重複する場合は本計画に一本化する。  

**指示があるまで修正は行わない。**
