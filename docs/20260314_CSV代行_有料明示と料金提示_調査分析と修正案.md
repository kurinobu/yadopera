# CSV一括登録代行：有料明示・料金提示 調査分析と修正案

**作成日**: 2026年3月14日  
**目的**: 「有料」の明示不足と料金の未提示により依頼が来ない・無料と誤認される事態を防ぐため、調査分析と大原則に準拠した修正案をまとめる。  
**実施結果**: 2026-03-14 に実装・デプロイ・ステージングブラウザテスト完了（§5 参照）。

---

## 1. 調査分析

### 1.1 指摘された問題

| 問題 | 内容 |
|------|------|
| 有料の明示不足 | ヘルプでは「有料オプション」と出るが、マニュアル側に有料の説明がなく、無料と思って依頼してくる可能性が高い。 |
| 料金がどこにも出ていない | 画面上で料金表・料金の提示がなく、「料金表はマニュアル等をご参照」とあるがマニュアルにも料金表はない。 |
| 文言 | 「代行をご希望の方はこちら」→ 有料であることが伝わりにくい。「CSV一括登録 代行の申し込み」タイトルも同様。 |

### 1.2 現状の「料金」の参照先

| 場所 | 内容 | 問題点 |
|------|------|--------|
| **アプリ内マニュアル（Manual.vue 4.8）** | 「料金表に沿った見積もり」「料金表や詳細は運営までお問い合わせください」と記載のみ。**料金表そのものは非表示。** | 利用者は金額を画面で確認できない。 |
| **申し込みページ（CsvBulkRequest.vue）** | 「料金は件数に応じてお見積もりします（**料金表はマニュアル等をご参照ください**）」 | マニュアルに料金表がないため、実質「参照先なし」。 |
| **ヘルプチャット** | 「有料オプション」「CSV一括登録の代行をお申し込みの方はこちら」 | 有料とは書いてあるが、金額・料金表は一切なし。 |
| **運営向けドキュメント** | `docs/20260307_プロジェクト現況と今後の計画_総括.md` §4.1、`CSV_bulk_registration_function_implementation_plan2.txt` に料金表あり。 | 利用者向けではなく、依頼が来るかどうかには直結しない。 |

**結論**: 利用者が「いくらか」「有料であること」を画面上で確実に把握できる場所がなく、無料と誤認したり、依頼を躊躇したりする状態になっている。

### 1.4 料金表の正（Single Source of Truth）

運用上の正は **総括 §4.1** および **plan2.txt**。

- **Standard**（最大100件）: 〜30件 ¥12,000 ／ 〜60件 ¥22,000 ／ 〜100件 ¥32,000  
- **Premium**（件数無制限）: 〜50件 ¥25,000 ／ 〜100件 ¥45,000 ／ 〜200件 ¥68,000 ／ 200件超 個別見積（¥90,000〜）  
- **料金に含まれる**: 日本語FAQの自然さ・文法確認、軽微な表現修正、AI翻訳＋品質確認、管理画面への登録作業、表示・動作チェック（初回）。  
- **料金に含まれない**: FAQの新規企画・構成設計、内容の事実確認、大幅な書き直し・再編集、法的表現監修・翻訳証明。

## 2. 大原則との対応

（出典: `docs/20260307_プロジェクト現況と今後の計画_総括.md` §0.1）

| 大原則 | 本件での適用 |
|--------|--------------|
| 根本解決 > 暫定解決 | 文言で「有料」を明示し、**料金を画面に提示する**ことで、無料誤認・依頼が来ない事態を根本から防ぐ。 |
| シンプル構造 > 複雑構造 | 料金表は「マニュアル本文＋申し込みページ」の2か所に同じ内容を載せ、表形式が難しければ箇条書きで統一。ManualContent.vue に新フォーマットを増やさない。 |
| 具体的 > 一般 | 「料金表はマニュアル等をご参照」ではなく、**金額を具体的に記載**する。 |
| 拙速 < 安全確実 | 依頼前に料金と有料であることを確実に伝え、トラブルを防ぐ。 |

### 3.2 料金の提示（どこに何を載せるか）

**方針**: 利用者が「有料であること」と「おおよその金額」を、**マニュアル**と**申し込みページ**の両方で確認できるようにする。  
料金表の正は総括 §4.1 のままとし、画面表示はその要約を「同じ内容」で2か所に載せる。

#### A. マニュアル（Manual.vue 4.8「有料代行をご希望の場合」ブロック内）

- **現状**: 「料金表に沿った見積もり」「料金表や詳細は運営までお問い合わせください」のみで、金額非表示。
- **修正**: 上記の直前に、**料金の目安**を箇条書きで追加する（ManualContent.vue はリストのみ対応のため、表ではなく箇条書き）。
  - 例（案）:
    - **料金の目安（税別）**: Standard 〜30件 ¥12,000、〜60件 ¥22,000、〜100件 ¥32,000。Premium 〜50件 ¥25,000、〜100件 ¥45,000、〜200件 ¥68,000、200件超は個別見積（¥90,000〜）。料金に含まれる内容・含まれない内容の概要を1行ずつ記載。
  - または、行数を抑える場合は「Standard 〜30件 ¥12,000〜、Premium 〜50件 ¥25,000〜。詳細は申し込みページの料金表をご確認ください。」とし、詳細は申し込みページに集約する案も可。

#### B. 申し込みページ（CsvBulkRequest.vue）

- **現状**: 「料金は件数に応じてお見積もりします（料金表はマニュアル等をご参照ください）」のみ。
- **修正**:
  1. **「申し込み前にご確認ください」ブロックの直後**（またはフォーム直前）に、**「料金の目安」セクション**を新設する。
  2. 内容: 総括 §4.1 と同じ表を、HTMLの `<table>` で表示（Vue テンプレート内で固定表でよい）。  
     - 見出し例: 「料金の目安（税別）」  
     - 表: プラン | 件数 | 料金  
     - その下に 1〜2 行で「料金に含まれる」「料金に含まれない」の要約を記載。
  3. 「希望登録件数」の補足文から「料金表はマニュアル等をご参照ください」を削除し、「上記の料金目安をご確認のうえ、件数をご記入ください」などに変更する。

#### C. ヘルプチャット

- 現時点では「有料オプション」の文言に加え、**「料金は申し込みページ・マニュアル4.8をご確認ください」**の一文を追加する程度に留める案とする（料金表の二重管理を避けるため、詳細はマニュアル・申し込みページを参照させる）。

## 4. まとめ

- **問題**: 有料の明示不足と、料金が画面上のどこにも提示されていないため、無料と誤認したり依頼が来にくい状態になっている。
- **対応**: (1) 全導線で「代行」を「代行（有料）」に統一する文言変更。(2) マニュアル4.8 と申し込みページの両方に、総括 §4.1 と整合した**料金の目安**を掲載する。
- **大原則**: 根本解決（料金を実際に表示）、シンプル（既存のマニュアル・ページ構造の範囲で対応）、具体的（金額を明記）に沿って修正する。
