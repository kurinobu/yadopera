# マニュアル更新・改訂 必読

**対象**: 管理画面の「ご利用マニュアル」を **更新・改訂する際は、作業前に本ドキュメントを必ず読むこと**。  
**目的**: 編集内容がブラウザに正しく反映されず「読みにくい」「反映されていない」事態を防ぐための再発防止策の徹底。

---

## 1. 必読である理由（過去の事象）

- マニュアルの文言・レイアウトを変更しても、**コードを変えただけではブラウザ表示が変わらない**ことがある。
- 原因は主に次の2点だった:
  1. **フロントの再ビルド・再起動をしていない** → 実行中のコンテナ／dev サーバーが古いバンドルを配信し続ける。
  2. **PWA の Service Worker とブラウザキャッシュ** → サイトデータ消去・SW 解除・ハードリロードをしないと古いアセットが表示され続ける。
- さらに **コンテンツの記述ルール**（箇条書きの行頭など）を守らないと、**表示側のリスト判定が効かず、箇条書きにならない**事象が発生した。
- 上記を「毎回丁寧に指示・説明しないと反映できない」事態として整理し、**本必読文書と手順の徹底**で同じミスを防ぐ。

---

## 2. 対象ファイルと役割

| ファイル | 役割 |
|----------|------|
| `frontend/src/views/admin/Manual.vue` | マニュアルの **本文**。`sections` の `content`（テンプレートリテラル）に章・節ごとのテキストを記載する。 |
| `frontend/src/components/admin/ManualContent.vue` | 本文を **HTML に変換して表示**。`formatContent()` で Markdown 風の記法（`**太字**`、行頭 `- ` のリスト、`<span class="manual-purpose">` など）を解釈し、スタイルを当てる。 |

- 文言の追加・変更は **Manual.vue** の `content` を編集する。
- 表示ルールの変更（箇条書きの出し方、目的ブロックのスタイルなど）は **ManualContent.vue** の `formatContent` とスタイルを編集する。

---

## 3. 期待する表示と動作（仕様）

以下が「正しく反映された」状態の仕様。**読みやすさのため、必ずこの表示になること**を前提に編集する。

| 項目 | 期待する表示・動作 |
|------|-------------------|
| **最重要ラベル** | 「★ FAQの登録（最重要）」「★ QRコードの発行（最重要）」などは **他より目立つ色（アンバー／黄系）** で表示される。 |
| **目的・重要な理由** | 各最重要項目の直下にある説明文は **青系の背景・左ボーダーのブロック** で囲まれ、通常の段落と明確に区別される。 |
| **箇条書き** | 「20件の初期デフォルトFAQを…」「基本情報のFAQ…」「設備・サービスのFAQ…」などは **「・」付きの箇条書き** として表示される。文章の塊ではなく、**1行1項目で「・」が先頭に付く**。 |

- これらが満たされていない状態は **「反映されていない」「読みにくい」** とみなす。

---

## 4. コンテンツ記述ルール（Manual.vue の content 内）

編集時に守ること。守らないと表示が意図どおりにならない。

### 4.1 箇条書きにしたい行

- **行頭に半角ハイフン＋スペース `- ` を付ける。**
- 表示では `ManualContent.vue` がリストとして解釈し、先頭に「・」を付けて表示する。
- 例（正）:
  ```
  - 20件の初期デフォルトFAQを確認して編集・削除・追加する
  - 基本情報のFAQ（チェックイン/アウト時間、WiFi情報など）を確認・編集する
  ```
- 例（誤）: 行頭に `- ` や `・ ` を付けずに本文だけ書くと、**箇条書きにならず塊の文章のまま**表示される。

### 4.2 目的・重要な理由のブロック

- 説明文を `<span class="manual-purpose">…</span>` で囲む。
- 表示では青系の背景・左ボーダーのブロックとして表示される。

### 4.3 最重要の見出し

- 見出し文中で「★ …（最重要）」としたい部分は `**★ …（最重要）**` のように **太字** で書く。
- `ManualContent.vue` 側で `.manual-important` に変換され、色付きで表示される。

## 5. 編集後の必須手順（反映を確実にする）

コードを変更しただけでは表示は変わらない。**以下をすべて実施すること。**

### 5.1 フロントの再ビルド・再起動

- **Docker 利用時**
  ```bash
  docker-compose up -d --build frontend
  ```
- **npm で開発サーバー利用時**
  ```bash
  cd frontend
  npm run build
  # 開発サーバーなら一度止めてから
  npm run dev
  ```
- キャッシュを疑う場合は `docker-compose build --no-cache frontend` のあと `docker-compose up -d frontend` も検討する。

### 5.2 ブラウザでキャッシュと Service Worker の解除

1. 開発者ツール → **Application** タブ → **Storage** で **「サイトデータを消去」** を実行。
2. **Service Workers** で登録されている SW があれば **Unregister** する。

### 5.3 ハードリロード

- **macOS**: `Cmd + Shift + R`
- **Windows / Linux**: `Ctrl + Shift + R`  
またはタブを閉じて `http://localhost:5173/admin/manual` をあらためて開く。

### 5.4 確認チェックリスト

- [ ] フロントを再ビルド・再起動した
- [ ] Application → Storage で「サイトデータを消去」した
- [ ] Service Worker を Unregister した
- [ ] ハードリロード（またはタブを閉じて再表示）した
- [ ] マニュアル 1.4 で「★ …（最重要）」が色付きで表示される
- [ ] 目的ブロックが青系の枠で表示される
- [ ] FAQ/QR の項目が「・」付きの箇条書きで表示される
