## 2. インボイス制度で必要なこと（要約）

- **適格請求書**（インボイス）または **適格簡易請求書**（簡易インボイス）のいずれかで、仕入税額控除の要件を満たす。
- 領収書は、制度で定める記載要件を満たせば **適格請求書として認められる**（請求書と領収書の両方は不要で、どちらか 1 つで可）。
- **必須の前提**: 発行側（やどぺら運営）が **適格請求書発行事業者** として登録済みであり、**登録番号（T＋13桁）** を取得していること。
- Stripe では、請求書を「適格請求書」、領収書を「適格簡易請求書」として扱う設定が可能。いずれも **登録番号を Stripe に設定** し、請求書・領収書に表示されるようにする必要がある。

参照: Stripe「領収書発行に際するインボイス制度への対応」、Stripe「日本で請求書を設定するためのベストプラクティス」。

## 3. Stripe ダッシュボード（PC）での設定手順（実態に即した説明）

以下は **Stripe ダッシュボード（ブラウザ・PC）** の実際のメニュー構成に沿った操作です。テストモード・本番モードで同じメニュー名ですが、左上の「テストモード」切替で環境が異なります。

**ナビの注意（2026-03-30 実測）**: **左サイドバーに「設定」という文言が常に出るわけではない**（ホーム・請求・顧客などの業務メニューが主）。設定配下へは **画面上部の歯車アイコン**、または **アドレスバーで `.../settings/...` に置換**（例: いま `.../test/settings/billing/...` にいるなら `.../test/settings/emails`）で遷移すると迷いにくい。

### 3.1 前提

- 適格請求書発行事業者としての **登録番号（T＋13桁）** を取得済みであること。
- 表示・請求書に載せる **事業者名・住所** を決めておくこと。

### 3.2 公開ビジネス情報（事業者名・住所）

1. Stripe ダッシュボードにログインし、左上で **テストモード／本番**（サンドボックス利用時は **サンドボックス**）を選択する。
2. **設定**へ進む（**上部ナビの歯車**、または URL で `.../settings/...` を開く）。
3. **「ビジネス」** 相当の一覧から **「ビジネスの詳細」**（Business details）を開き、**「公開情報」** で **「顧客向けの表示情報」** を編集する。  
   - 実測 URL 例（サンドボックス・`acct` 付き）: `https://dashboard.stripe.com/acct_<ACCOUNT_ID>/test/settings/business-details`  
   - 英語ドキュメントでいう **Public details** は別パス（`.../settings/public`）のこともある。**日本語 UI では「ビジネスの詳細」内の「公開情報」で揃えた**（2026-03-30）。
4. **法人名（日本法人）**は行により挙動が分かれる。**1行目・2行目が非活性**で「サンプル株式会社」「サンプルカブシキガイシャ」のまま、**3行目のみ編集可**になる場合がある。**3行目に漢字の会社名を入れるとバリデーションで拒否され、ローマ字の法人名（例: `aplauso Co., Ltd.`）で保存に成功した**（2026-03-30・サンドボックス）。根拠: Stripe ドキュメント上、日本向けに **kanji/kana 系と別表記**が求められる説明がある（引き継ぎ書 `docs/20260330_セッション引き継ぎ_Stripe_インボイス_ダッシュボード作業.md` セクション 3 のリンク）。
5. **公開情報**の住所・サポート連絡先を入力し保存する。

Stripe の請求書・領収書に載る事業者表記は、**公開情報・ブランディング・サンドボックスのプレースホルダ**等の組み合わせになりうる。`docs/20260330_セッション引き継ぎ_Stripe_インボイス_ダッシュボード作業.md` に **領収 PDF とホストページで名称が一致しない事例**を記録済み。

### 3.3 請求書の設定（登録番号・納税者番号）

1. **「設定」** → **「請求」**（Billing）の **「請求書」**（Invoice）を開く。  
   - URL 例: `https://dashboard.stripe.com/settings/billing/invoice`  
   - サンドボックスでは `https://dashboard.stripe.com/acct_<ACCOUNT_ID>/test/settings/billing/invoices/general` のように **`invoices/general` 配下**になる実測あり（2026-03-30）。
2. **「税」**（Tax）**専用タブ**がある場合はそこを開く。**2026-03-30 の UI では「一般」タブの縦長ページ内に「請求書の税務情報」セクション**があり、そこで **JP TRN をデフォルト**にできた。
3. **日本用の納税者番号（JP TRN）** を追加する。
   - 登録番号は「T＋13桁」を、Stripe が求める形式で入力する（種別で「JP TRN (登録番号)」を選ぶ）。
   - 追加後、画面右下の **「保存」** をクリックする。
4. **【必須】デフォルトとして設定する**  
   納税者番号を「追加しただけ」では請求書 PDF に表示されない。**すべての請求書・領収書 PDF に自動で載せるには「デフォルト」の指定が必要**である。
   - 追加した納税者番号の行の右側にある **「⋯」（オーバーフローメニュー）** をクリックする。
   - メニューから **「デフォルトとして設定」**（Set as default）を選択する。
   - 画面右下の **「保存」** を再度クリックする。
   - 設定後、その納税者番号に「デフォルト」などのラベルが付いていれば成功。以降に**確定される**請求書から登録番号がヘッダーに表示される。
5. 請求書の番号付け方法やデフォルトの税設定など、必要に応じて他項目も確認する。  
   - 日本向けベストプラクティスでは、**小計後に端数処理するように税金を設定**することが推奨される（請求書の設定ページ内で税設定を確認）。

この設定により、**Stripe が発行する請求書（Invoice）** に適格請求書発行事業者の登録番号が記載される。

### 3.4 領収書（適格簡易請求書）に登録番号を表示する

1. **「設定」** → **「メール」**（Emails）を開く。  
   - URL 例: `https://dashboard.stripe.com/settings/emails`（サンドボックスは `https://dashboard.stripe.com/acct_<ACCOUNT_ID>/test/settings/emails`）
2. **顧客向けメール** のうち **領収書（Receipt）** に関する設定を探す。
3. **領収書メールに登録番号を記載する**（およびこれに相当する英語ラベル）を**オン**にする。  
   - **実測（2026-03-30・YadOPERA サンドボックス）**: 当該トグルが **非活性のまま**操作できない。**請求書レンダリング用テンプレートを新規作成（有効化）した後も非活性のまま**だった。直下の案内どおり **請求書テンプレート設定**を開いても、**メール側トグルが有効化される保証は本計画書の時点ではない**。公開ドキュメントで「非活性の条件」を特定するには Stripe サポートまたは本番 UI での再確認が必要。
4. **デフォルトの言語**を **日本語**にした（2026-03-30）。領収・請求メールのローカライズに効くが、**法人名 3 行目の「漢字不可」バリデーションを緩める効果はない**（別フィールドの話）。
5. 納税者番号そのものは **3.3** で登録した JP TRN が参照される。  
   - **実測**: 上記メールトグルが触れない状態でも、**領収書 PDF に `登録番号 T8140001032079` が表示される**事例があった（同一領収書番号の PDF を再取得した際に住所表記も更新された）。**メール本文に載せる設定**と **PDF の表示**は切り分けて確認すること。

これで、Stripe が生成する **領収書** に適格請求書発行事業者の登録番号が載り、適格簡易請求書として利用しやすくなる、**を目指す**。3.4 のトグルが環境により使えない場合は **3.5・税理士確認・Stripe サポート**を参照。

### 3.5 確認方法

- ステージング（テストモード）で、有料プランの施設として少額でも課金が発生するようにし、請求書が発行されたあとで:
  - 管理画面 **「プラン・請求」**（フロントのパスは **`/admin/billing`**。例: `https://yadopera-frontend-staging.onrender.com/admin/billing`）→ 請求履歴 → **「領収書を表示」** をクリックして、Stripe の Hosted Invoice Page または PDF を開く。
- 表示内容に **事業者名・登録番号（T＋13桁）・取引内容・税区分・金額** などが含まれていることを確認する。
- **同一行の請求の PDF を再ダウンロードしただけでは「新規確定後の検証」に相当しない場合がある**（一部フィールドのみ更新された事例の報告あり）。**3.6 原因 B** に従い、**設定保存のあとに新しく確定・支払い済みになった請求**で確認するのが安全。
- Stripe ダッシュボードで **手動の請求書**を使う場合、公式は請求書エディタで **「Review invoice」→「Send the invoice」** の流れと説明している（[Use the Dashboard](https://stripe.com/docs/invoicing/dashboard)）。**`/invoices/in_xxx`（`/edit` なし）だけでは「Finalize」と英語で書かれた操作が見えない**ことがある。**詳細と編集は URL で切り替える**（`.../in_<ID>` と `.../in_<ID>/edit`）。状態遷移の定義は [Status transitions and finalization](https://docs.stripe.com/invoicing/integration/workflow-transitions)。
- 不備があれば、下記「3.6 請求書・領収書に登録番号が表示されない場合」を参照し、上記 3.2〜3.4 の設定と、Stripe の日本向けドキュメント（適格請求書／適格簡易請求書の記載要件）を再確認する。

**詳細な作業ログ・URL 表**: `docs/20260330_セッション引き継ぎ_Stripe_インボイス_ダッシュボード作業.md`

### 3.6 請求書・領収書に登録番号が表示されない場合（原因と対処）

**原因 A: 納税者番号を「デフォルト」にしていない**

- Stripe では、納税者番号を**追加しただけでは**請求書 PDF に表示されない。**「デフォルトとして設定」** し、**保存**する必要がある。
- **対処**: **3.3** の手順 4 を実施する。請求書の設定 → 税タブ → 該当の納税者番号の「⋯」→ **「デフォルトとして設定」** を選択 → **保存**。設定後、その番号に「デフォルト」のラベルが付くことを確認する。

**原因 B: 確認している請求書が「デフォルト設定より前に確定」されている**

- Stripe の仕様で、**請求書が確定（finalized）したあとでは、アカウントの納税者番号を追加・変更・削除しても、その請求書の PDF には反映されない**。確定時点でデフォルト納税者番号が取り込まれる。
- **対処**: 「デフォルトとして設定」を保存した**あと**に新しく確定された請求書で確認する。例: 次回のサブスクリプション請求が発行されたあとで「領収書を表示」を開く、またはテスト用に新しい請求書を発行して PDF を確認する。過去に確定済みの請求書（例: Invoice-E7U4Y5NE-0002）は、設定を変えてもその PDF には登録番号は出ない。

**原因 C: 別のタブ・別の設定ページである**

- 納税者番号は **「設定」→「請求」→「請求書」** の **「税」（Tax）タブ** で追加・デフォルト設定する。請求書の「全般」タブだけでは不十分な場合があるので、**税タブ**で「請求書の税務情報」を開き、上記 A・B を確認する。

**参照**: Stripe ドキュメント [Account and customer tax IDs with Invoicing](https://docs.stripe.com/tax/invoicing/tax-ids)（「Setting default tax IDs」「You can't add, change, or remove account tax IDs after an invoice is finalized」）。

## 3.7 請求書・領収書の表示言語を日本語にする（推奨）

領収書・請求書が英語だと読みづらいため、**顧客（施設）向けの表示言語を日本語に寄せる**。

### 3.7.1 Stripe ダッシュボードで既存顧客の言語を日本語にする

1. 左メニュー **「顧客」** を開き、対象顧客を選択する。
2. 顧客詳細ページで **「更新」**（Update details / Edit）を開く。
3. **Language（言語）/ Preferred language（優先言語）** の項目を **Japanese（日本語）** に設定して保存する。

この設定により、Stripe は **請求書メール・請求書PDF・領収書メール・領収書PDF** 等を日本語にローカライズする（Stripe の仕様）。

### 3.7.2 アプリ側で今後作成される顧客の既定言語を日本語にする

本リポジトリでは、Stripe Customer 作成時に `preferred_locales=["ja"]` を付与するよう実装を更新済み（`backend/app/services/stripe_service.py`）。
これにより、**今後アプリが作る Stripe 顧客は日本語が既定**になる。

### 3.7.2.1 既存顧客を一括で日本語にする（推奨・手動作業ゼロ）

日本以外に顧客がいない前提なら、Stripe ダッシュボードで顧客ごとに言語を変更するのは非効率。
本リポジトリに **一括更新スクリプト**を追加したため、Docker から1回実行して完了できる。

- スクリプト: `backend/scripts/set_stripe_customers_preferred_locale_ja.py`
- 実行例（まずは dry-run 推奨）:

```bash
docker compose run --rm backend python scripts/set_stripe_customers_preferred_locale_ja.py --dry-run
docker compose run --rm backend python scripts/set_stripe_customers_preferred_locale_ja.py
```

（`STRIPE_SECRET_KEY` が環境変数として backend コンテナに渡っている前提。必要なら `-e STRIPE_SECRET_KEY=...` で渡す。）

### 3.7.3 注意（過去の請求書に反映されない）

- 請求書PDFは **確定（finalized）時点の情報**で生成されるため、言語設定や納税者番号の設定を変更しても、**既に確定済みの請求書PDFには反映されない**。
- 変更後は **新しく発行・確定される請求書**で日本語表示になっていることを確認する。

## 4. develop で設定するか・本番だけで設定するか（推奨方針）

### 4.1 結論: **develop（ステージング）の Stripe テストモードで先に設定し、手順を確立したうえで、本番稼働時に本番 Stripe で同様の設定を行う**

### 4.2 理由

1. **拙速 < 安全確実（大原則）**  
   本番でいきなり登録番号や領収書設定を行うより、先にテストモードで同じ手順を実施し、領収書プレビューで登録番号・事業者名が正しく表示されることを確認してから本番で行う方が安全である。

2. **手順の確立とドキュメント化**  
   テストモードのダッシュボードで実際に画面を開きながら、「設定」→「公開ビジネス情報」「請求書」「メール」の各項目を埋める手順を実施し、本計画書や手順書に反映できる。本番では同じ手順を本番モードで繰り返すだけにできる。

3. **Stripe の仕様**  
   テストモードと本番モードは別環境のため、テストモードで行った設定は本番には反映されない。そのため、「develop で設定すると本番に影響する」という心配はなく、ステージング側で試すことで本番設定のリハーサルになる。

4. **運用の一貫性**  
   これまで Stripe は develop（ステージング）で設定・検証し、本番導入は別フェーズで行う方針である（総括・Phase 4 計画）。領収証のインボイス対応も「ステージングで設定・確認 → 本番で同様に設定」と揃えると分かりやすい。

### 4.3 本番だけで設定する場合のデメリット

- 本番ダッシュボードで初めて操作することになり、入力ミスや見落としのリスクが増える。
- 手順書をテストモードで検証していないため、説明と実際の画面の差異が残りやすい。
- 大原則の「安全確実」「具体的（実態に即した説明）」にやや反する。

### 4.4 実施の流れ（推奨）

| 順序 | 環境 | 実施内容 |
|------|------|----------|
| 1 | 事業者側 | 適格請求書発行事業者の登録を完了し、登録番号（T＋13桁）を取得する。 |
| 2 | Stripe **テストモード**（develop 用） | 3.2〜3.4 の設定を行う。公開ビジネス情報・請求書の納税者番号・領収書の納税者番号表示を有効化。 |
| 3 | ステージング（develop） | テスト課金で請求書を発行し、管理画面から「領収書を表示」で内容を確認。登録番号・事業者名等が表示されることを確認。 |
| 4 | ドキュメント | 上記手順を本計画書または別の「領収証インボイス対応 設定手順書」に固定し、本番実施時に流用する。 |
| 5 | Stripe **本番**（本番稼働時） | 本番用 Stripe アカウントで 3.2〜3.4 と同様の設定を実施。本番用の事業者名・登録番号で設定。 |

## 7. まとめ

- **目的**: 管理画面から提供している領収書（Stripe の Hosted Invoice / PDF）を、インボイス制度に合わせて適格請求書または適格簡易請求書として使えるようにする。
- **やること**: ① 事業者として登録番号を取得、② Stripe ダッシュボードで公開ビジネス情報・請求書の納税者番号・領収書の納税者番号表示を設定、③ ステージングで表示確認してから本番で同様に設定する。
- **推奨方針**: **develop（ステージング）の Stripe テストモードで先に設定し、手順と表示を確認したうえで、本番稼働時に本番 Stripe で同じ設定を行う。** 本番だけで設定するより、安全確実で手順の再現性が高い。
