# 3a | 開発者ダッシュボードにメールアドレス表示を追加 — 実装計画

**作成日**: 2026年3月9日  
**対象**: 計画「3a」開発者ダッシュボードにメールアドレス表示を追加  
**基準**: 要約定義書 v0.3、アーキテクチャ設計書 v0.3、20260307_プロジェクト現況と今後の計画_総括、大原則

---

## 1. 目的・スコープ

- **目的**: 開発者管理ダッシュボードの「施設一覧」に、各施設のメールアドレスを表示する。
- **スコープ**: 3a のみ（3b「宿泊施設名・メールアドレス一覧のCSVダウンロード」は別タスク）。
- **表示するメール**: 施設の連絡先メール（`facilities.email`）。既存カラムのためDBマイグレーションは不要。

---

## 2. 大原則との対応

| 原則 | 対応 |
|------|------|
| 根本解決 > 暫定解決 | 施設マスタの正の情報源（facilities.email）を表示する。 |
| シンプル構造 > 複雑構造 | 既存の施設一覧API・テーブルに1列追加するだけとする。 |
| 統一・同一化 > 特殊独自 | 既存の FacilitySummary / 施設一覧テーブルのパターンに従う。 |
| 具体的 > 一般 | 変更ファイル・手順を本計画に明示する。 |
| 拙速 < 安全確実 | Docker で動作確認してからマージする。 |
| Docker環境必須 | すべての修正・テストは Docker 環境で実行する。 |

---

## 3. コードベース調査結果

### 3.1 現状

- **バックエンド**
  - `GET /api/v1/developer/stats/facilities` が施設一覧を返す。
  - `FacilitySummaryResponse`: id, name, is_active, plan_type, faq_count, chats_7d, errors_7d, last_admin_login のみ。email は含まれていない。
  - `Facility` モデルに `email = Column(String(255), nullable=False)` が既に存在（`backend/app/models/facility.py`）。
- **フロントエンド**
  - `DeveloperDashboard.vue`: 施設一覧テーブルに「施設名」「状態」「料金プラン」「FAQ数」「チャット（7日）」「エラー（7日）」「最終ログイン」の7列。メールアドレス列なし。
  - `FacilitySummary` 型（`frontend/src/types/developer.ts`）に email なし。

### 3.2 ブランチ・デプロイ

- **ブランチ戦略**（要約定義書 v0.3.1）: `main`（本番）、`develop`（ステージング）、`feature/*`（開発）。
- **本タスク**: `develop` から `feature/developer-dashboard-email` を作成し、完了後に `develop` へマージする。

### 3.3 Docker

- 検証は `docker-compose up -d` で起動した環境で実施する（README・大原則に準拠）。

## 4. 実装ステップ

### Step 1: バックエンド — スキーマとAPI

| 項目 | 内容 |
|------|------|
| ファイル | `backend/app/schemas/developer.py` |
| 変更 | `FacilitySummaryResponse` に `email: str` を追加。 |
| ファイル | `backend/app/api/v1/developer/stats.py` |
| 変更 | `FacilitySummaryResponse` 構築時に `email=facility.email` を渡す。 |

- DB・マイグレーション: 変更なし（既存 `facilities.email` を利用）。

### Step 2: フロントエンド — 型とUI

| 項目 | 内容 |
|------|------|
| ファイル | `frontend/src/types/developer.ts` |
| 変更 | `FacilitySummary` に `email?: string` を追加。 |
| ファイル | `frontend/src/views/developer/DeveloperDashboard.vue` |
| 変更 | 施設一覧テーブルに「メールアドレス」列を追加（施設名の次など）。表示は `{{ facility.email || '-' }}`。 |

### Step 3: Docker での動作確認

1. `docker-compose up -d` で起動。
2. 開発者ログイン → ダッシュボードで施設一覧を表示し、メールアドレス列が表示されることを確認。
3. 必要に応じて Swagger (`/docs`) で `GET /api/v1/developer/stats/facilities` のレスポンスに `email` が含まれることを確認。

**ブラウザテスト即実行用**: 手順・URL・チェックリストは **`docs/開発者ダッシュボード_メールアドレス表示_Step3_Docker動作確認とブラウザテスト準備.md`** を参照。

### Step 4: マージ

- `feature/developer-dashboard-email` を `develop` にマージ。
- ステージングデプロイ後、開発者ダッシュボードで再確認推奨。

## 6. リスク・注意

- **個人情報**: 開発者画面は開発者認証必須のため、メールアドレスは認証済み運用者のみが閲覧する。既存の施設一覧と同様に扱う。
- **null**: スキーマ上 `facilities.email` は `nullable=False` のため通常は空でない。フロントでは `facility.email || '-'` で未設定時は `-` 表示とする。
