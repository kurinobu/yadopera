## 2. Render.com Static Siteサービスのログ確認方法

### 2.1 ログの確認場所

**方法1: Logsタブ**
1. Render.comダッシュボードで`yadopera-frontend-staging`サービスを選択
2. 左側のメニューから「**Logs**」タブをクリック
3. デプロイ時のビルドログが表示される

**方法2: Eventsタブの詳細**
1. 「Events」タブを開く
2. 各デプロイイベントをクリック
3. 詳細画面でログを確認

**方法3: Monitorタブ**
1. 「Monitor」タブを開く
2. 「Logs」セクションを確認

### 2.2 デプロイログがない場合の確認

**確認項目**:
1. **デプロイの状態**: イベントページで各デプロイの状態を確認（成功/失敗/進行中）
2. **ビルドコマンド**: Settingsタブでビルドコマンドが正しく設定されているか確認
3. **環境変数**: Environmentタブで環境変数が正しく設定されているか確認

---

## 3. デプロイが実際に実行されていない可能性

### 3.1 考えられる原因

**原因1: デプロイが失敗している**
- ビルドエラーが発生している可能性
- ログが記録されていない可能性

**原因2: デプロイが実際には開始されていない**
- イベントは表示されているが、実際のデプロイプロセスが実行されていない
- `render.yaml`の設定が正しく読み込まれていない可能性

**原因3: Static Siteサービスが`render.yaml`を使用していない**
- 手動で作成されたStatic Siteサービスが`render.yaml`を参照していない
- ダッシュボードの設定が優先されている

### 3.2 確認すべき項目

**確認項目1: デプロイの状態**
- イベントページで各デプロイイベントの状態を確認
- 「成功」「失敗」「進行中」などの状態を確認

**確認項目2: Settingsタブの設定**
- Build Command: `npx vite build`が設定されているか
- Publish Directory: `dist`が設定されているか
- Root Directory: `frontend`が設定されているか

**確認項目3: Environmentタブの設定**
- `VITE_API_BASE_URL`が設定されているか
- `VITE_ENVIRONMENT`が設定されているか

## 4. 次のステップ

### 4.1 即座に確認すべき項目

1. **Logsタブを確認**
   - `yadopera-frontend-staging`サービスの「Logs」タブを開く
   - 最新のログを確認

2. **Eventsタブの詳細を確認**
   - 各デプロイイベントをクリック
   - 詳細画面でログやエラーメッセージを確認

3. **デプロイの状態を確認**
   - 各デプロイイベントの状態（成功/失敗/進行中）を確認

### 4.2 確認結果に基づく対応

**ケース1: ログが表示されている場合**
- ログの内容を確認
- エラーメッセージがあれば、原因を特定

**ケース2: ログが表示されていない場合**
- デプロイが実際には開始されていない可能性
- Settingsタブの設定を確認
- 手動デプロイを試行

**ケース3: デプロイが失敗している場合**
- エラーメッセージを確認
- 原因を特定して修正
