# GA4 統計日次確認手順（運用版）

最終更新: 2026-04-22  
対象: `https://yadopera.com/` の LP A/B 計測

---

## 1. この手順書の目的

毎日または任意のタイミングで、Owner が GA4 上で以下を同じ手順で確認できるようにする。

- 全体イベント数（`lp_ab_view` / `lp_ab_cta_click` / `lp_ab_assign`）
- `variant` 分解（`a` / `b` / `(not set)`）
- 全体CTRおよび A/B別CTR（分解可能な場合）

---

## 2. 事前情報（固定）

- プロパティ: やどぺら（測定ID `G-BE9HZ0XGH4`）
- 使う画面: `レポート > ユーザー エンゲージメントと… > イベント`
- 表内検索文字列: `lp_ab_`
- 分解ディメンション: `variant`（カスタム）

---

## 3. 毎回の確認手順（最短）

1. GA4 で対象プロパティを開く（`G-BE9HZ0XGH4`）。
2. `レポート > ユーザー エンゲージメントと… > イベント` を開く。
3. 右上の日付を設定（推奨: 過去7日 または 任意の集計期間）。
4. 表内検索に `lp_ab_` を入力し、3イベントを表示:
   - `lp_ab_view`
   - `lp_ab_cta_click`
   - `lp_ab_assign`
5. `イベント名` 列の右にある `+` を押し、`カスタム > variant` を選択。
6. 表で `イベント名 × variant` の行を読み取る。
7. 下記テンプレに件数を転記する。

---

## 4. 記録テンプレ（コピペ用）

```
確認日:
確認期間:

lp_ab_view: a=?, b=?, (not set)=?
lp_ab_cta_click: a=?, b=?, (not set)=?
lp_ab_assign: a=?, b=?, (not set)=?

全体CTR: lp_ab_cta_click総数 / lp_ab_view総数 = ?%
CTR_A: click_a / view_a = ?%   （view_a=0 なら算出不可）
CTR_B: click_b / view_b = ?%   （view_b=0 なら算出不可）

判定メモ:
```

---

## 5. 判定ルール（迷わないため）

1. `a` または `b` の表示数が 0 の場合、A/B比較は未成立。
2. `(not set)` が過半数の場合、A/B勝敗判定は保留する。
3. まずは「全体イベント送信が継続しているか」を優先確認する。
4. A/B採用判断は、少なくとも `a` と `b` の双方で十分な表示母数を確保してから実施する。

---

## 6. 詰まりやすいポイント

- `今日` のみだと 0 件になることがある。期間を広げて再確認する。
- イベント詳細画面で `variant` カードが出ないことがある。その場合はイベント一覧表で分解する。
- `DebugView` が見えなくても、イベント一覧で件数が出ていれば送信は継続している。

---
