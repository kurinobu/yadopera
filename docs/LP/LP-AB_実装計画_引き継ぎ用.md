## 5. A/B 表示仕様（実装方針）

1. 初回表示時、`sessionStorage` にキーが無ければ A/B をランダム決定。
2. 決定値を `sessionStorage` に保存（例: `lp_ab_variant`）。
3. 同一タブ内では保存値を優先し、同じ面を表示。
4. A/B のどちらか一方のみ表示（もう一方は非表示）。
5. フッターは共通化し、法務リンク差分が生じない構造にする。

---

## 6. GA4 計測仕様（実装方針）

既存の `gtag` 設定を維持し、以下を追加する想定。

- 面の露出イベント（例: `lp_ab_view`）
  - パラメータ: `variant`（`a` / `b`）
- CTAクリックイベント（例: `lp_ab_cta_click`）
  - パラメータ: `variant`（`a` / `b`）, `cta_id`（任意）

最小要件:

- A/B別の表示数（露出数）が取れる
- A/B別のCTAクリック数が取れる

## 12.1 追加の確認結果（2026-04-22）

期間 `2026-04-21〜2026-04-22`、GA4 イベント画面で `variant` 分解を再確認した結果:

- `lp_ab_view`: `(not set)=15`, `b=1`, `a=0`
- `lp_ab_cta_click`: `(not set)=12`, `b=0`, `a=0`
- `lp_ab_assign`: `(not set)=7`, `b=1`, `a=0`

判定:

1. イベント送信そのもの（`lp_ab_view` / `lp_ab_cta_click` / `lp_ab_assign`）は継続して記録されている。
2. ただし `variant` は大半が `(not set)` で、A/B比較（A/B別CTR判定）は未成立。
3. この状態のままでも「全体イベント数」は継続記録されるが、「A/B別の確定比較」はできない。

補足:

- 全体CTR（未分解）: `12 / 16 = 75%`
- `b` は母数 1 のため評価に使えない。

## 13. 再発防止ルール（運用手順）

- 事実確認は「口頭の記憶」ではなく、**最新スクショを正**とする。
- UIラベルは環境差があるため、案内時は必ず「今見えている表示名」を優先する。
- 1ステップずつ案内し、完了レスポンスを受けてから次へ進む。
- 計測確認で詰まった場合は DebugView 固執を避け、標準レポート（イベント）で確認する。
- GA4確認の具体手順は `docs/LP/GA4_AB_表示数_クリック数_率_確認手順.md` を正本とする。
- 日次運用の確認手順は `docs/LP/GA4_統計日次確認手順_運用版.md` を参照する。
