## 2. 確認項目（どこで何を確認するか）

**結論（迷わないように）**

- **「FAQが30件入る」「RAGが動く」** → **ブラウザ（ステージングのアプリ）** でだけ確認する。Render の Logs は使わない。
- **「埋め込みAPIが登録時に呼ばれていない」** → **Render の Backend の Logs** でだけ確認する。ブラウザでは分からない。

| 何を確認するか | どこで確認するか |
|----------------|------------------|
| FAQが30件入っている（2.1） | **ブラウザ**：管理画面のFAQ一覧 |
| RAGが動いている（2.2） | **ブラウザ**：ゲストチャットで質問して回答を見る |
| 埋め込みAPIが登録時に呼ばれていない（2.3） | **Render の Logs**：Backend → Logs タブでメッセージを確認 |

---

### 2.1 新規登録でFAQが投入されること → ブラウザで確認

1. ステージングの**新規登録画面**で、テスト用の施設を1件登録する（メール認証が必要なら認証まで実施）。
2. ログイン後、**管理画面のFAQ管理**で、その施設のFAQが **30件** 登録されていることを確認する。
3. または API（施設トークンで `GET /api/v1/admin/faqs`）で件数が 30 であることを確認する。

### 2.2 RAG検索が期待どおり動くこと → ブラウザで確認

1. ゲスト向けチャット（または該当施設のチャット画面）を開く。
2. 初期FAQの内容に沿った質問（例：チェックイン時間、Wi‑Fi、静粛時間など）を送信する。
3. FAQに基づいた回答が返ることを確認する。

### 2.3 埋め込みAPIが登録時に呼ばれていないこと → Render の Logs で確認

**開く場所**: Render ダッシュボード → 対象プロジェクト → **Backend（API 用 Web Service）** → **「Logs」タブ**。

**いつ見るか**: 新規登録を実行した**直後**。その時刻の前後に出ているログ行を見る。

**ログで見る内容（2.3 の確認だけ。30件やRAGはログでは見ない）**:

- **出ていればOK**: `"Using precomputed embedding"` が多数ある。
- **出ていなければOK**: `"Generating FAQ translation embedding"` が**無い**（登録処理のあいだに出ていないこと）。
- **出ていたらNG**: `"Preset embedding missing"` の WARNING は無いこと。
