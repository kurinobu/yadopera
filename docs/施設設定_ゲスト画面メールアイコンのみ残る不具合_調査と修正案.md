# ゲスト画面：メール非表示時にアイコンのみ残る不具合 — 調査分析と修正案

**作成日**: 2026-03-03  
**現象**: 施設設定で「ゲスト画面にメールアドレスを表示する」のチェックを外すと、メールアドレス文字は表示されなくなるが、**メールのアイコンのみが残る**。電話番号の場合は、電話番号とアイコン両方とも表示されなくなる。

**参照**: 大原則（Phase3 現況と残存課題・実装計画）、施設設定メールアドレス_ゲスト表示スイッチ_実装計画

---

## 1. 原因調査

### 1.1 現状の実装

**FacilityHeader.vue**（ゲスト向け施設ヘッダー）では、電話とメールの表示は次のように実装されている。

- **電話番号**（7–12行目）: 親要素 `<div v-if="facility.phone">` でアイコン＋テキストをまとめて表示制御。`facility.phone` が falsy のときはブロック全体が描画されない。
- **メール**（13–18行目）: 同様に親要素 `<div v-if="facility.email">` でアイコン＋`<span>{{ facility.email }}</span>` をまとめて表示制御している。

```vue
<div v-if="facility.phone" class="flex items-center space-x-2">
  <svg>...</svg>
  <span>{{ facility.phone }}</span>
</div>
<div v-if="facility.email" class="flex items-center space-x-2">
  <svg>...</svg>
  <span>{{ facility.email }}</span>
</div>
```

テンプレート上は、電話・メールとも「親 div に v-if を付け、値がないときはブロックごと非表示」という同じ構造になっている。

### 1.2 想定される原因

理論上は `show_email_on_guest_screen = false` のときバックエンドが `email: null` を返し、`v-if="facility.email"` が false になってブロック全体が非表示になるはずである。それにもかかわらず「アイコンのみ残る」ということは、次のいずれかが考えられる。

| 要因 | 内容 |
|------|------|
| **A. 値が null ではなく空文字・空白** | API やストア経由で `email` が `""` や `" "` になっている場合、JavaScript では `""` は falsy だが、一部環境・変換の影響で `" "` などが入ると `v-if` が true になり得る。その場合、`{{ facility.email }}` は空白のみで見た目は「アイコンのみ」になる。 |
| **B. レスポンスのキー欠落・型のずれ** | 公開APIが `email` を省略したり、別形式で返したりしていると、フロントで `undefined` 以外の値（例: 空文字）が入る可能性がある。 |
| **C. 表示条件の甘さ** | 現状は `v-if="facility.email"` のみ。null/undefined は弾けるが、「空文字」「空白のみ」を明示的に「表示しない」としていないため、それらが入った場合にブロックが表示され得る。 |

いずれにせよ、**「表示すべきメールが無いときは、アイコンも含めてブロック全体を確実に非表示にする」**ように条件を強化することが必要である。

### 1.3 バックエンド・データフローの確認

- **公開API**（`GET /api/v1/facility/{slug}`）: `FacilityService.get_facility_public_info` で `show_email = getattr(facility, "show_email_on_guest_screen", True)` とし、`show_email` が False のとき `email=None` で返している。通常は JSON で `"email": null` になる。
- **フロント**: Welcome.vue / LanguageSelect.vue / Chat.vue で `facilityApi.getFacility(...)` の `response.facility` を `facilityStore.setFacility(response.facility)` でストアに保存し、FacilityHeader は `facility` をストアから参照している。

通常はここまでで `email` が null になる想定だが、キャッシュ・別経路・将来的な仕様変更で「空文字や空白」が入る可能性を考慮し、フロントの表示条件を「値が実質あるときだけ表示」に寄せるのが安全である。

---

## 2. 大原則に沿った修正方針

- **根本解決 > 暫定対応**: 「表示すべきメールが無いときはブロック全体（アイコン＋テキスト）を出さない」を条件で保証する。
- **統一・同一化**: 電話番号と同じく「表示すべき値が無ければブロックごと非表示」にし、電話とメールの挙動を揃える。
- **拙速 < 安全確実**: null/undefined に加え、空文字・空白のみも「非表示」とみなす条件にする。

---

## 3. 修正案

### 3.1 対象ファイル

- `frontend/src/components/guest/FacilityHeader.vue`

### 3.2 変更内容

**方針**: メール・電話とも「表示するのは、値が存在しかつ空白以外の文字があるときだけ」とする。  
そのために、**表示条件を「文字列として trim した結果が空でない」**に統一する。

1. **メール**
   - 現状: `v-if="facility.email"`
   - 変更: `v-if="(facility.email ?? '').toString().trim() !== ''"`
   - 効果: `null` / `undefined` / `""` / `" "` のいずれでもブロック（アイコン＋span）が描画されない。

2. **電話番号（挙動の統一）**
   - 現状: `v-if="facility.phone"`
   - 変更: `v-if="(facility.phone ?? '').toString().trim() !== ''"`
   - 効果: メールと同様に「実質中身がない」場合はブロック全体を非表示にし、電話とメールの挙動を同一にする。

**v-if の付く要素**: これまで通り、**アイコンとテキストを包む親の `<div>` にのみ**付与する。子の `<span>` には付けない（現状もそうなっているため、そのまま維持する）。

### 3.3 修正後のテンプレート例

```vue
<div v-if="(facility.phone ?? '').toString().trim() !== ''" class="flex items-center space-x-2">
  <svg class="w-4 h-4" ...>...</svg>
  <span>{{ facility.phone }}</span>
</div>
<div v-if="(facility.email ?? '').toString().trim() !== ''" class="flex items-center space-x-2">
  <svg class="w-4 h-4" ...>...</svg>
  <span>{{ facility.email }}</span>
</div>
```

### 3.4 補足

- バックエンドは現状のまま（`show_email_on_guest_screen` が False のとき `email: null` を返す）でよい。
- 上記変更はフロントのみで完結し、API 契約やストアの型を変えずに済む。
- 電話・メールで同じ条件にすることで、「非表示時はアイコンも出さない」という仕様が一貫する。
