## 3. 調査した公式情報（事実：出典付き）

- **サンドボックスで「アカウントをコピー」した場合**: **プレースホルダー用のビジネスデータ**が入る旨が [Sandbox settings](https://docs.stripe.com/sandboxes/dashboard/sandbox-settings) に記載。
- **日本の確認情報**: 法人名の **`name_kana` / `name_kanji`** 等がセットで前提となる旨が [Required verification information](https://docs.stripe.com/connect/required-verification-information) の日本向け説明にある（Connect 文脈。標準アカウント UI との 1 対 1 は本文書では断定しない）。
- **請求書の状態遷移**: `draft` → **finalize** → `open` → 支払いで `paid` 等。[Status transitions and finalization](https://docs.stripe.com/invoicing/integration/workflow-transitions)  
  **下書きのままでは支払い収集はできない**旨が同ページに記載。
- **ダッシュボードで請求書を作成・送付する流れ**: 請求書エディタで内容入力後 **「Review invoice」** し **「Send the invoice」**する手順が [Use the Dashboard](https://stripe.com/docs/invoicing/dashboard) に記載。**Finalize というラベルのボタンが主手順として説明されているわけではない**。
- **領収書のカスタマイズ**: ブランディング・**Public details** が関係する旨が [Email receipts and paid invoices](https://docs.stripe.com/payments/checkout/receipts) に記載。

## 4. 本セッションでの指引きの失敗・混乱要因（事実）

- **「左サイドバーで設定」**という説明が、利用者の画面では **左サイドに「設定」文字がなく**実際と不一致だった。**設定周りは画面上部の歯車アイコン、または既に `.../settings/...` に入っている場合は URL での移動**が実態に合った（利用者フィードバック）。
- **「§3.5」等の略**で、**どのファイルのどの節か**が一瞬で分からなくなった（利用者指摘）。以後は **`docs/領収証_インボイス対応化_設定計画.md` の「3.5 確認方法」**のように**ファイル名＋見出し**で書く。
- **Finalize / 請求書を確認** など**英語ドキュメント由来のラベル**を、そのまま日本語 UI に当てはめて案内し、**ボタンが見つからない**状態を招いた。
- **3.4 のメールトグル**を「テンプレ作成で必ず直る」とは限らないまま進めた結果、**テンプレ作成後も非活性**で**未解決**（利用者実測）。
