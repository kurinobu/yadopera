# マニュアル（Manual.vue）編集後の反映手順

**※ マニュアルを更新・改訂する際は、まず [docs/マニュアル更新・改訂_必読.md](./マニュアル更新・改訂_必読.md) を必読のこと。**

**対象**: 管理画面の「ご利用マニュアル」（`frontend/src/views/admin/Manual.vue` および `ManualContent.vue`）を編集したあと、ブラウザに変更が表示されない場合の対処です。

**注意**: 「デプロイ後FAQ未更新」などで言及される「FAQ未更新」は **ヘルプチャット用の operator_faq（DB）** の話であり、本マニュアル（静的コンテンツ）とは別問題です。

---

## 期待する表示と動作（仕様）

以下が「正しく反映された」状態の仕様です。読みやすさのため、必ずこう表示されること。

| 項目 | 期待する表示・動作 |
|------|-------------------|
| **最重要ラベル** | 「★ FAQの登録（最重要）」「★ QRコードの発行（最重要）」は **他より目立つ色（アンバー／黄系）** で表示される。通常の黒太字ではない。 |
| **目的・重要な理由** | 各最重要項目の直下にある説明文（登録数が多く…／ゲストが質問にアクセスする入口です…）は **青系の背景・左ボーダーのブロック** で囲まれ、通常の段落と明確に区別される。 |
| **箇条書き** | 「20件の初期デフォルトFAQを…」「基本情報のFAQ…」「設備・サービスのFAQ…」などは **「・」付きの箇条書き** として表示される。文章の塊ではなく、1行1項目で「・」が先頭に付く。 |

- これらが満たされていない状態は **「反映されていない」「読みにくい」** とみなす。

---

## 結果の説明と評価（キャッシュ消去後も反映しない場合）

- **現象**: サイトデータ消去・強制リロード・再ログイン後も、上記「期待する表示と動作」にならない。★が色付きでない、目的が青ブロックでない、箇条書きが「・」付きで表示されず塊のまま。
- **評価**: ブラウザキャッシュ・Service Worker 以外に原因がある。
  - **想定原因**: (1) リスト行の先頭文字（・など）の文字コード差でリスト判定が効いていない、(2) 「★…（最重要）」の置換正規表現が一致していない（全角/半角括弧など）、(3) `v-html` 内要素に scoped スタイルが当たっていない。
- **対応**: `ManualContent.vue` でリスト判定の正規表現を緩和（・/·/• など複数文字種・任意空白）、★置換を全角・半角括弧両対応に変更、`.manual-important` / `.manual-purpose` を scoped 外のスタイルでも定義して v-html 内に確実に適用する。

---

## 原因

- フロントは **Vite PWA（Workbox）** で Service Worker が有効です。
- **precache** により JS/CSS がキャッシュされるため、ビルドし直してもブラウザが古いアセットを返すことがあります。
- 開発時でも、一度読み込んだあとでは **Service Worker が古いバンドルを返し続ける** ため、編集が「反映しない」ように見えます。

## 反映手順（実施順）

### 1. フロントを再ビルドして起動し直す

**Docker 利用時**

```bash
docker-compose up -d --build frontend
```

**npm で開発サーバー利用時**

```bash
cd frontend
npm run build
# 開発サーバーなら一度止めてから
npm run dev
```

### 2. ブラウザでキャッシュと Service Worker を解除する

1. **開発者ツール** を開く（F12 または 右クリック → 検証）。
2. **Application** タブを開く。
3. 左の **「Storage」（ストレージ）** を選択。
4. **「サイトデータを消去」**（Clear site data）を実行。
   - 「サードパーティのCookieを含む」は必要に応じて。
5. 左の **「Service Workers」** を開き、登録されている SW があれば **「Unregister」** で登録解除。
6. 開発者ツールを閉じる。

### 3. ハードリロードで開き直す

- **Windows / Linux**: `Ctrl + Shift + R`
- **macOS**: `Cmd + Shift + R`

または、タブを閉じて `http://localhost:5173` をあらためて開く。
