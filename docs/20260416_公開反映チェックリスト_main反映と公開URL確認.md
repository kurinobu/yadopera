# develop→main 反映と公開URL確認チェックリスト

最終更新: 2026-04-16  
目的: 「develop では修正済みだが、公開サイト（main）に未反映」を防止する。

---

## 1) develop 側の完了確認

- `git checkout develop`
- `git pull --ff-only origin develop`
- `git status` がクリーンであることを確認
- 反映したいコミットが `develop` に入っていることを確認
  - 例: `git log --oneline -n 10`

---

## 2) main への反映

- `main` が別 worktree の場合は、その運用に従って反映する
- 通常運用の基本手順（例）
  - `git checkout main`
  - `git pull --ff-only origin main`
  - `git merge --no-ff develop`
  - `git push origin main`

補足:
- コンフリクトが出たら、解消後に `git status` で差分を確認してから push
- `--force` は使わない

---

## 3) 「main に入ったか」の機械確認

- `git fetch origin`
- `git log --oneline --decorate --max-count=5 origin/main origin/develop`
- 対象コミットが `origin/main` 側から辿れることを確認

必要ならファイル単位で確認:
- `git show origin/main:<path/to/file>`

例（privacy の電話番号削除確認）:
- `git show origin/main:landing/legal/privacy.html`

---

## 4) 公開URL最終確認（必須）

### 4-1. 表示確認

- トップ: `https://yadopera.com/`
- 主要ページ:
  - `https://yadopera.com/about.html`
  - `https://yadopera.com/legal/privacy.html`
  - `https://yadopera.com/legal/terms.html`
  - `https://yadopera.com/legal/operation.html`
  - `https://yadopera.com/legal/faq.html`
  - `https://yadopera.com/legal/security.html`
  - `https://yadopera.com/legal/tokusho.html`

### 4-2. キャッシュ切り分け

- ブラウザでハードリロード（`Cmd + Shift + R`）
- それでも古い場合はクエリ付きURLで確認
  - 例: `https://yadopera.com/legal/privacy.html?v=20260416-1`

---

- スマホ表示: iPhone 1端末 / Android 1端末
