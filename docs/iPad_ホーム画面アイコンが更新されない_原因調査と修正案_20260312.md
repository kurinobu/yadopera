# iPad でホーム画面アイコンが更新されない 原因調査と修正案

**作成日**: 2026年3月12日  
**事象**: デプロイ・ブラウザテスト完了後、Android ではアイコン表示されるが、iPad ではアイコンが更新されない。  
**基準**: 大原則（根本解決・シンプル・統一・具体・安全確実・Docker必須）に準拠した修正案とする。

---

## 1. 原因調査

### 1.1 想定される要因

| 要因 | 説明 | 該当度 |
|------|------|--------|
| **iOS の apple-touch-icon キャッシュ** | iOS Safari は「ホームに追加」時に取得したアイコンを **URL 単位で強くキャッシュ**する。同一 URL (`/pwa-192x192.png`) のままデプロイで中身だけ差し替えても、iOS はキャッシュを返し再取得しない。 | **高** |
| **Service Worker によるアイコン配信** | `vite.config.ts` の Workbox `globPatterns: ['**/*.{js,css,html,ico,png,svg}']` により、ビルド時に **pwa-*.png がプリキャッシュ**される。iPad で古い SW がまだ有効な間は、古いプリキャッシュからアイコンが返り、新アイコンが見えない。 | **中** |
| **apple-touch-icon のサイズ** | Apple は **180×180** を推奨。現状は 192×192 を流用しており動作はするが、iOS が別サイズを優先している可能性は低い（主因はキャッシュ）。 | 低 |

### 1.2 根拠

- **同一 URL のままでは更新されない**: 公開情報・Stack Overflow 等で、`apple-touch-icon` に **クエリパラメータ（例: `?v=2`）を付与して URL を変える**と、iOS が別リソースとみなして再取得し、アイコンが更新されることが報告されている。
- **Workbox**: プリキャッシュ対象に `png` が含まれるため、ビルド成果物の `pwa-192x192.png` / `pwa-512x512.png` は SW に含まれる。デプロイ後に SW が更新されれば新しいアイコンが配られるが、**URL が同じ限り、既に「ホームに追加」済みの端末では iOS 側キャッシュが優先**され、更新が遅れる。

### 1.3 結論（根本原因）

**アイコン URL が固定（`/pwa-192x192.png` 等）のため、デプロイでファイルを差し替えても iOS がキャッシュを返し、新しいアイコンを取得していない。**

根本対応は **「アイコン URL にキャッシュ無効化用のクエリ（例: `?v=版番号`）を付け、デプロイごとに URL を変える」** こと。

## 2. 大原則に沿った修正方針

| 原則 | 適用 |
|------|------|
| **根本解決 > 暫定解決** | キャッシュ無効化（URL の変更）により、デプロイ後の再追加で確実に新アイコンが取得されるようにする。 |
| **シンプル構造 > 複雑構造** | 版番号は 1 箇所で管理し、index.html と manifest の両方で同じ値を使う。 |
| **統一・同一化** | アイコン URL の付与ルールを index と動的 manifest で揃える。 |
| **具体的 > 一般** | 版番号の定義場所・参照場所・アイコン差し替え時の手順を明示する。 |
| **拙速 < 安全確実** | 変更後は Docker ビルド・実機（iPad）での「ホームに追加」→ アイコン確認を行う。 |

## 3. 修正案

### 3.1 アイコン URL にキャッシュ無効化クエリを付与する

- **目的**: デプロイのたびに（またはアイコン差し替えのたびに）URL を変え、iOS に再取得させる。
- **方法**: 版番号（例: `2` または日付 `20260312`）をクエリで付与する。  
  - 例: `/pwa-192x192.png?v=2`  
  - アイコンを差し替えたら版番号を 1 つ上げる（次回は `?v=3`）。

### 3.2 版番号の一元管理（統一）

- **案 A（推奨・シンプル）**: 定数を 1 ファイルで定義し、manifest と index の両方で参照する。
  - `frontend/src/utils/iconVersion.ts` を新設し、`export const ICON_VERSION = '2'` を定義。
  - `manifestGenerator.ts`: 上記を import し、`src: \`/pwa-192x192.png?v=${ICON_VERSION}\`` のように使用。
  - `frontend/index.html`: ビルド時に上記値を埋め込む必要があるため、**Vite の HTML 変換**で対応する（下記 3.4）。

- **案 B（より簡易）**: 版番号を 2 箇所に書く（`index.html` の link と `manifestGenerator.ts` の icon src）。アイコン差し替え時に両方を同じ値に更新する旨をドキュメントに明記。  
  - 実装が簡単だが、「1 箇所だけ更新し忘れる」リスクがある。

**推奨**: 案 A（1 箇所定義＋Vite で index に注入）。

### 3.3 動的 manifest の icon src 変更

- **ファイル**: `frontend/src/utils/manifestGenerator.ts`
- **変更**: `iconVersion.ts` の `ICON_VERSION` を import し、各 icon の `src` を  
  `'/pwa-192x192.png?v=' + ICON_VERSION`  
  `'/pwa-512x512.png?v=' + ICON_VERSION`  
  に変更する。

### 3.4 index.html への版番号注入（案 A の場合）

- **方法**: Vite の `transformIndexHtml` フックで、プレースホルダ（例: `__ICON_VERSION__`）を `iconVersion.ts` の値（または環境変数）に置換する。
- **index.html**:  
  - 例:  
    `<link rel="apple-touch-icon" href="/pwa-192x192.png?v=__ICON_VERSION__" />`  
  - ビルド後に `?v=2` のように置換される。
- **vite.config.ts**: `plugins` に 1 本、`transformIndexHtml` で `__ICON_VERSION__` を置換する処理を追加する（`iconVersion.ts` を読むか、`define` で渡した値を使う）。

### 3.5 （任意）iOS 向け 180×180 アイコンの追加

- Apple は **180×180** を推奨。確実性を上げるなら、既存の生成スクリプトで 180 を追加し、  
  `<link rel="apple-touch-icon" sizes="180x180" href="/pwa-180x180.png?v=__ICON_VERSION__" />`  
  を追加する。
- 必須ではないが、**統一**のため 192/512 と同様に「版番号付き URL」にする。

### 3.6 アイコン差し替え時の手順（具体化）

1. `scripts/generate_pwa_icons_circular.py` で新しいアイコンを生成（必要なら 180 も追加）。
2. `frontend/public/` に pwa-192 / pwa-512（と pwa-180）を上書き。
3. `frontend/src/utils/iconVersion.ts` の `ICON_VERSION` を 1 つ増やす（例: `'2'` → `'3'`）。
4. Docker でフロントビルド・動作確認。
5. ステージングへデプロイし、iPad で「ホームに追加」し直してアイコンを確認。
